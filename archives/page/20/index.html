장고 템플릿 문법 1

장고에서 화면을 렌더링 하기 위해서는 템플릿 엔진을 사용한다. 템플릿 엔진은 각자 자기만의 템플릿 문법을 제공하는데 여러가지 기본 문법과 태그 필터를 살펴본다

  1. 컨텍스트 변수 출력하기
    • 뷰에서 컨텍스트 변수로 값을 전달해준다면 해당 변수 이름을 가지고 출력할 수 있다
  1. 폼 출력하기
    Create, Update 뷰 같은 경우 폼을 출력한다. 폼 태그들을 출력할 때 p, li, table 등의 태그로 감싸진 형식으로 출력되게 만들 수 있다. 또 입력 폼에서는 csrf_token은 필수 이다.

이메일 서버 세팅하기

  1. 외부에서 메일 보내기 서버를 사용하려면 gmail의 설정을 변경해야 한다. gmail에 로그인해서 화면 우측 상단에 있는 설정 버튼을 클릭한다

  1. [전달 및 POP/IMAP] 탭으로 이동해 IMAP 사용에 체크하고 [변경사항 저장]버튼을 누른다.

  1. 보안수준이 낮은 앱의 액세스 허용을 위해 계정 -> Google 계정을 클릭해 계정 설정으로 이동한다.

  1. 계정 설정 화면에서 [보안] 탭을 클릭한다. 해당 탭에서 [보안 수준이 낮은 앱의 액세스] 부분을 찾아 [액세스 사용 설정] 버튼을 클릭한다.

  1. [보안 수준이 낮은 앱 허용]을 활성화 한다.
  1. 장고 프로젝트의 settings.py에 메일 서버를 설정한다.
  1. 메일 보내기를 테스트 해본다

아마존 인증서 발급 받기

Domain

  • naver.com

sub Domain

  • blog.naver.com
  • static.web.nvaer.com

이메일 검증이 편하긴 하다(1년마다 갱신 해야함)
DNS검증을 하게되면 자동으로 갱신이 된다.

Certificate Manager로 들어간다

[인증서 요청]을 클릭하게되면 위와 같은 화면이 나온다
*(와일드카드)로 발급받게되면 서브도메인 전체에 대한 적용이 가능하다

검증방법에서는 DNS 검증을 선택하고 [검토]버튼을 누른다

검증 방법과 신청 내용을 확인하고 [확인 및 요청]버튼을 클릭한다

도메인 옆의 삼각형을 누르면 상세적보가 나오는데 [Route 53에서 레코드 생성] 버튼을 클릭한다
레코드 내용을 확인하고 [생성] 버튼을 클릭한다

생성 성공 메시지를 확인하고 [계속]버튼을 클릭한다

다시 인증서 목록으로 돌아오면 검증 미완료 상태를 확인할 수 있고
시간이 지나면 검증 완료가 되고 인증서가 발급된다
