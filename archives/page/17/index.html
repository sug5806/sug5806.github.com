2019-05-17 Django에 Bootstrap 적용하기

2019/05/17 카테고리 Django

bootstrap 적용하기

  1. base.html에 bootstrap 설치하기

    • CDN 스크립트로 설치하기 : 리소스 절감효과, 최신버전을 자동 유지, CDN을 사용하면 접속 속도가 증가

    • 다운로드해서 설치하기 : 커스터마이징 가능, 도메인을 넘나들지 않아도 된다.(특정 위치에서는 더 빠를 수 있다)

    • jQuery Slim 버전은 ajax 기능이 빠져있다.

      • min 버전 : 용량을 최소화한 버전 -> 띄어쓰기 생략, 엔터 생략
      • beautify : min 버전은 소스코드를 알아보기 힘들기 때문에 beautify해서 다시 보기 좋게 만든다.

    • 모든 static 리소스들은 캐시가 걸린다. -> 수정사항 바로 반영되지 않음

      1. 웹 서버에서 캐시타임 조절 -> 자주 수정되는 경우 캐시 타임을 확 줄인다.(보통 1~2일 유지)
      2. 이미 받은것은 만료가 될때까지 갱신이 안됨 -> static 리소스의 파일명을 변경한다.(가장 쉬운 방법이지만 장고의 소스코드를 바꿔야하므로 귀찮은 방법)
      3. static 리소스 뒤에 Query String을 붙인다. -> 아예 캐싱이 안되도록 한다.(뒤에 Query string을 붙이면 다른 파일이라고 인식하여 다운받음)

    • HTML 파일에 적용하기

      • CSS 파일은 헤더 바로밑에
      • JS파일은 /body태그 바로위에 위치시킨다 - jQuery, popper, bootstrap 순으로 위치시킨다

기존에는 user-agent로 모바일, 데스크탑, 태블릿으로 구분을 했으나 현재에는 반응형 웹을통해 화면의 너비를 통해 기종 구분없이 화면이 달라진다

Grid system : Bootstrap에서 지원하는 내장 반응형웹

  • col-로 쪼갠 나머지를 다른 애들이 먹는다
  • 여백을 col로 지정만 해주고 내용 부분만 조절하면 된다.

    • 여러개를 통합해서 한번에 물어보느냐 아니면 짧게 여러번 물어보느냐가 중요하다
  • 코드레벨 검색
  1. 검색어
    1. 검색어를 제대로 받았냐
    2. filter 메소드
      • 어떤 항목(제목에서, 글 내용에서, 작성자에서)
      • 어떤 옵션(대소문자 구분, Hello로 시작하냐 끝나냐 등등)
      • filter(field 이름) : 매칭
        • Document.objects.filter(pk=25, title=?)을 하면 and로 한다 Ex) pk=25이고 title이 ?인것
      • filter(field 이름__옵션) : 옵션으로매칭
        • Document.objects.filter(title__startwith=“4”) : title이 4로 시작하는 것
        • i를 붙여주면 ignorecase가 된다
        • filter(title__icontains=“4”) : 중간에 4가 들어있느냐
        • Forignkey Field는 fileter(category__name=‘4’)로 한다
        • filter(category__name__endswith=‘4’) : 카테고리의 이름이 4로 끝나느냐

from django.db.models import Q

  • default_q = Q(title__icontains=‘1’)
  • second_q = Q(title__icontains=‘1’)
  • default_q | second_q
  • 섞어서 쓰려면 q객체를 다 쓴다음에 일반 필드를 적어주면 된다

select_related -> DB에 부담을 줄 것이냐? 아니면 서버에 부담을 줄것이냐?
Document.objects.select_related(‘author’).all()

  1. select_related -> Join Query를 만들어서 한큐에 데이터를 불러온다.
    • ForeingnKey까지만 묶을 수 있다.
    • Join
      • author를 가져오는데 inner_join을 통해 DB access를 줄인다.
      • inner_join : 서로 가지고 있는 데이터만 합쳐서 보여준다 -> 교집합
      • outer_join : 서로 가지고 있는 데이터와 함께 어떤 테이블을 기준으로 레코드를 가져오는데 없으면 null로 가져온다
  2. prefetch_related -> Join을 사용하지 않음
    • Document.objects.prefetch_related(‘author’).all()
    • Document를 부르고, Category를 부러서 DB내에서 붙여서 하지않고 파이썬 코드에서 병합한다.
    • ManyToMany 이런것도 지원함
    • selete_related로 할수 없는 것은 prefetch_related를 사용한다.

select_related, prefetch_related를 사용하지 않는 경우
참조 테이블에 대한 데이터 질의를 항상 실행한다. = DB에 대해 access를 항상하여 부담 증가

QuerySet 캐시

장고 내에도 캐시가 있어 한번 질의를 실행하면 캐시에 저장한다.

  • object_list = Document.objects.all() : 쿼리가 실행되지 않음
  • object_list[:4] : 4번까지 꺼내온다
    • LIMIT을 걸어 slicing을 하게되면 캐싱이 되지않는다
    • 한번 전체가 돌고 난뒤에 slicing을 하면 캐싱이 된다
    • 한번 다 돌고 평가가 되어야 캐싱이 일어난다

object_list = Document.objects.all() -> 캐싱 아님
[object.title for object in object_list] -> 캐싱

object_list = Document.objects.all()
object_list[4] -> 캐싱 아님
object_list[5] -> 캐싱 아님

object_list = Document.objects.all()
[object.title for object in object_list]
object_list[4] -> 캐싱
object_list[5] -> 캐싱

~을 제외하여 가져오기

Document.objects.exclude(title__incontains=‘1’) : 타이틀중에 1이 포함된 것을 제외하고 가져오기

  • 필터를 이용했을때는 필드를 나열하면 자동으로 and연산이 되었지만 exclude는 안된다.

제외하고 싶은 카테고리를 먼저 추출하고 거기서 전체 카테고리에 제외하고싶은 카테고리가 있으면 exclude를 한다
Document.objects.exclude(category__in=Category.objects.filter(name__icontains=‘question’))
