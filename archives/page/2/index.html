Go 기초

Go의 키워드

Go는 키워드가 25개가 존재합니다

Go의 기본

Go는 기본적으로
main 패키지에서부터 실행을 시작하며
입,출력을 위해 fmt 패키지가 필요합니다

x := 42 -> x를 선언하고 42를 할당합니다. (명시적 타입을 생략가능하고 선언과 동시에 할당하며 함수 밖에서는 사용할 수 없습니다)

x = 99 -> 위에서 선언한 x를 99로 재할당합니다.

출력하기

Print의 경우 바이트수와 발생한 쓰기 오류를 리턴하며
Sprint의 경우 결과 문자열을 리턴합니다.

타입 변환

다른 언어에서는 casting라고 하는것을 Go에서는 conversion이라 합니다.

iota

iota는 상수 선언에서 사용하며 처음 선언했을때 0으로 시작하며 자동으로 1씩 증가합니다.
같은 스코프 내에서 다시 할당해도 초기화 되지 않으며 다른 스코프에서 재할당을 하면 0으로 초기화됩니다.

2019-09-12 Docker 명령어1

2019/09/12 카테고리 Docker

개인적으로 공부한 내용을 정리하였습니다.

docker ps : 현재 실행중인 컨테이너 목록

  • a : 종료된 모든 컨테이너의 목록이 출력

docker rm ‘컨테이너 이름’ or ‘컨테이너의 ID’ : 해당 컨테이너를 삭제

docker stop ‘컨테이너 이름’ or ‘컨테이너의 ID’ : 실행 중인 컨테이너를 정지시킴

docker run ‘이미지이름’: 해당 이미지를 실행시킨다. 이미지가 없으면 pull해서 만듬

  • d : 백그라운드에서 실행시킴
  • sleep ‘숫자’ : 숫자만큼 실행이 유지됨(초 기준)
  • –name ‘원하는 이름’ : 원하는 이름으로 컨테이너 생성
  • ‘이미지이름:원하는 버전’ : 원하는 버전으로 실행 가능
  • -p A:B : A포트를 B포트에 연결하겠다.

docker rmi ‘이미지 이름’ : 해당 이미지를 삭제함

  • 이미지를 지우기 위해선 해당 이미지로 만든 컨테이너를 모두 삭제시켜야 이미지삭제 가능

컨테이너나 이미지를 지울때 ID를 하나씩 쳐서 지우는게 매우 귀찮은데 이럴때는 docker rm $(docker ps -a -q) 를 하면 전체 컨테이너가 모두 지워진다, 삭제뿐만 아니라 stop에도 사용가능

docker exec ‘컨테이너의 ID’ ‘명령어’: 컨테이너에서 명령어를 실행시킨다.

2019-09-11 리눅스 기본2(xargs, alias, ls)

2019/09/11 카테고리 리눅스

개인적으로 공부한 내용을 정리하였습니다.

표준 입력을 지원하지 않는 명령어(Ex. echo, rm 등)을 파이프를 통해 연결하고 싶다면 xargs 명령어를 통해 연결할 수 있다.

date | cut --delimiter=" " --fields=4 | echo 를 하면 echo는 표준 입력을 지원하지 않으므로 터미널에 출력되지 않지만 date | cut --delimiter=" " --fields=4 | xargs echo 을 하면 터미널에 출력된다.

자신의 home 디렉토리에 .bash_aliases 파일을 만들고 그안에 alias aliasName=“command1 -option args | command -option args…” 을 하게되면 별칭으로 설정한 것으로 명령을 실행할 수 있다.(설정 뒤 터미널 재시작 필수)

  • zshell을 사용할 경우 .zshrc에 적어주면 된다.
  • 적은 후 저장하고 source /home/.zshrc 를 하면 사용 가능하다.

~ : 현재 유저의 홈디렉토리

pwd : 현재 내가 위치하고 있는 디렉토리의 절대경로

ls : 현재 디렉토리의 내용을 출력한다

  • F 옵션을 사용하면 명시적으로 나타낸다.
  • l 옵션을 사용하면 자세한 정보가 나온다
  • h 옵션을 사용하면 사람이 읽기쉬운 상태로 용량이 출력된다.
  • a 옵션을 사용하면 숨김파일까지 모두 보여준다.
