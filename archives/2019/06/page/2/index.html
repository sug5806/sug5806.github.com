pip install django-extension을 설치합니다

이런 화면이 뜰텐데요. 일반 shell로하면 직접 import 해야하는것을 shell_plus로 하면 자동으로 import가 됩니다. 매우 편리합니다

그럼 이제 만든 자식1을 수정 해보도록 하겠습니다.

name이 자식1, age가 13이던 자식이 자식10, age가 20으로 변경되었습니다.

그럼 이제 생성한 자식을 삭제하도록 하겠습니다

2019-06-08 Django admin page 다루기3

2019/06/08 카테고리 Django

오늘은 admin page에서 수정과 link에 대해서 알아보겠습니다.

기존의 admin page입니다 여기서 나이나 주소를 수정하려면 일일히 들어가서 수정해야합니다. 매우 번거롭지 않나요? 이 문제를 해결하겠습니다.

admin.py를 수정합니다.

그런데 name을 변경하고 싶은데 할 수 없습니다. 왜냐하면, name에 link가 걸려있기 때문입니다. 이 문제를 해결하도록 하겠습니다.

admin.py를 수정합니다.

list_display에 id를 추가시켰습니다. id는 django에서 모델을 만들때 자동으로 만들어주는 primary key입니다.

그러나 이번엔 숫자가 너무 작아 클릭을 하기 힘듭니다. 이것을 수정하도록 하겠습니다

admin.py를 수정합니다.

문제가 생겼습니다. 앞서 말한 것과 같이 link를 클릭하려면 수정을 할 수 없습니다.

개인적인 의견으로는 name에 link를 걸지 않아도 모든 필드 값을 수정할 수 있으므로 link는 id만 거는 게 좋다 생각합니다.

Models.py

기본적인 모델입니다. 이름, 나이, 주소가 있습니다

settings.py, admin.py에서 방금 만든 앱을 등록하도록 하겠습니다. 앱이름은 admin_page입니다

./manage.py migrate 명령어를 실행하여 DB반영 합니다.

./manage.py createsuperuser 명령어를 통해 관리자 계정을 생성해 줍니다.


비밀번호가 너무 짧고 단조로워 오류가 뜨네요.
관리자 계정을 만들때는 어렵게 만들도록 합시다

이제 ./manage.py runserver를 통해 서버를 실행시킨후 http://127.0.0.1:8000/admin/로 접속해 관리자 페이지로 들어가도록 하겠습니다

보시면 만든 Admin page가 있습니다 들어가 봅니다.
들어가면 아무것도 없을텐데 하나 만들어 줍니다.

만들었는데 AdminPage object(1)이라고 뜹니다. 이렇게되면 구분을 할수 없습니다. models.py를 수정하여 제대로 출력해보도록 하겠습니다
