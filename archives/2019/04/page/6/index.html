2019-04-12 Tree

2019/04/12 카테고리 자료구조

Tree

트리는 그래프의 일종이다(Connected acyclic graph)

  • 그래프에서의 순회를 트리에서 적용할 수 있다

정의

  • 연결이 있고 사이클이 있는 그래프
  • 루트 노드(root)가 있다
  • 트리를 구성하는 노드 간에 단순 경로가 존재
    • 이동할때 갔던 경로를 다시 방문하지 않고 갈 수 있으면 단순경로가 있다고 한다
  • Start에서 출발해서 Start로 올수있다 (cyclic)
  • Connected : 경로가 무조건 하나는 있다

Tree 용어

  • 노드의 차수 : 자식의 갯수
  • 트리의 차수 : 노드의 차수 중에 가장 큰것
  • 리프 노드 : 자식이 없는 노드
  • 엣지 : 노드와 노드를 연결하는것
  • 트리의 높이 = 깊이 : 트리의 레벨
  • 트리가 되기위한 조건식 : 엣지 = 노드 - 1
  • 포레스트 : 서브 트리의 집합(Connected 만족 X) -> 트리가 모여있으므로 포레스트
  • 일반적으로 트리를 이야기 하면 이진트리(binary Tree)

Binary Tree

정의

  1. 어떤 노드의 최대 자식 수가 2개이하 인것
  2. 공집합 혹은 루트와 왼쪽 서브트리, 오른쪽 서브 트리로 이루어진 유한집합, 각각의 서브트리는 모두 이진 트리이다
    • 자식이 없지만 있다고 치는 노드를 공노드(Empty Node)라 한다.

특징

  • 각 Level에서의 최대 노드수 : 2^(level-1) 개
  • 트리 전체의 최대 노드 수 : 2^(Tree level) -1 개

포화 이진트리

  • 모든 레벨이 꽉 차 있다.

완전 이진트리

  • 일정 레벨 까지는 꽉 차있고 바로 밑 레벨부터 왼쪽에서 오른쪽으로 채워져있는 트리

편향 이진트리

  • 한쪽으로만 치우친 트리

트리의 순회

  • 재방문을 허용하지 않고 트리를 구성하는 모든 노드를 한번씩 방문하는것
순회의 계열은 2가지로 나뉜다

  • Stack계열

    • DFS(Depth First Search) - 그래프용어 : 깊이 우선 순회
      1. 전위순회(Preorder traversal) : A - B - C의 순서
      2. 중위순회(Inorder traversal) : B - A - C의 순서
      3. 후위순회(Post traversal) : B - C - A의 순서
    • 재귀 : 재귀를 하면 Stack Frame이 쌓임
    • 반복문 + Stack

  • Queue계열

    • BFS(Broath First Search) - 그래프용어: 너비 우선 순회
      • 트리에서는 레벨 순서 순회(level order)

BST(Binary Search Tree)

dictionary 구현하기

  1. hashing -> 파이썬의 dictionary는 hashing으로 구현됨
    • 테이블
  2. BST -> O(log N)
    • 이진트리의 일종
    • 모든 원소는 서로 다른 key를 가진다
    • 왼쪽 서브 트리에 있는 모든 키들은 루트의 키보다 작다.
    • 오른쪽 서브 트리에 있는 모든 키들은 루트의 키보다 크다.
    • 왼쪽 서브 트리와 오른쪽 서브 트리도 이진 탐색 트리이다.
    • 재귀적으로도 만족 해야한다 : 자식 트리역시 똑같은 로직으로 작동해야 한다

사족

  • delete() -> None : 리스트에서도 지워지고 메모리에서 지워짐
  • remove() -> removed data : 리스트에서 지워진 데이터를 반환(데이터를 반환 함으로써 사용자에게 어떻게 처리할건지 결정하게함)
  • python의 dictionary는 immutable 객체만 key로 사용할 수 있다
  • BST는 속도가 빠르기때문에 DB에서 사용하는 B-Tree역시 BST의 일종이다
  • BST는 Binary Tree의일종이므로 만들때는 상속으로 만드는게 좋다

2019-04-11 List와 Array의 차이점

2019/04/11 카테고리 자료구조

리스트(List)와 배열(Array)의 차이점

  1. 동적배열(Dynamic Array) vs 연결리스트(Linked List)의 차이

    • DynamicArray

      • 장점:
        • search : O(1) -> Indexing을 사용하기 때문
          • 메모리에 가져올때 지역성이 적용되어 캐시히트가 일어나기 때문
      • 단점:
        • Insert : O(n) -> 삽입을 하기위해 기존의 n개의 데이터를 뒤로 밀어야 한다
        • Delete : O(n) -> 삭제를 한 뒤 남아있는 데이터를 앞으로 당겨야 한다.

    • LinkedList

      • 장점: 데이터의 삽입, 삭제가 매우 빠르다
        • Insert : O(1)
        • Delete : O(1)
      • 단점:
        • Search : O(n)
          • 지역성이 적용될 확률이 낮기때문에 캐시히트가 일어나기 힘들다

Search를 많이하면 배열을, Insert, Delete를 할때는 연결리스트를 주로 사용한다

다이나믹 힙(Dynamic Heap)기법으로 메모리 할당을 한쪽으로 몰아도 되도록이면 배열을 사용한다

2019-04-10 Git 특강2 branch

2019/04/10 카테고리 기타

Git

브랜치란?

  • 분기점을 생성하고 독립적으로 코드를 변경할 수 있도록 도와주는 모델

  • Ex)

    • matser branch

    • another branch

  • 로컬의 branch와 리모트의 branch는 따로 존재한다

  • git branch -r : 리모트의 브랜치를 조회한다

  • git branch : 로컬의 브랜치를 조회한다

브랜치 생성

  • git branch feat/loop -> feat/loop라는 새로운 브랜치를 만든다
    • 현재 브랜치의 현재 상태 그대로 복제하여 만든다

브랜치 이동

  • git checkout feat/loop -> feat/loop 브랜치로 이동한다.

리모트 레포와 동기화

  • git push -u origin feat/loop : 리모트 레포에 feat/loop라는 새로운 브랜치 생성
    • -u : upstream의 약자 -> 리모트 레포에 feat/loop가 없으므로 해주어야 한다.

브랜치를 해도 deletion이 없으면 평행으로 되고 있으면 밑으로 추가되고, 우리가 논리적으로 생각하는 것으로 그림을 그린다.

브랜치 병합하기

  1. 우선 병합하고자 하는 브랜치로 이동한다 -> git checkout feat/loop
  2. git merge feat/con-state : feat/loop를 feat/con-state의 내용과 동기화시킨다(feat/con-state와 같아진다)
  3. 자동으로 commit이 생긴다.

브랜치 삭제하기

  • git branch -D/d 브랜치명
    • D는 강제로 삭제한다

브랜치및 머지를 보다 쉽게 하기

  • git flow 전략 : 다운로드 링크

    • master는 항상 개발이 끝난 것만 보여야한다

  • git init -> git flow init : git init을 하면서 flow까지 한꺼번에 함

  • develop 브랜치를 만들어도 기능 개발을 할때는 브랜치를 또 만들어서 해야한다

  • git flow feature start Name : Name의 브랜치를 생성하고 checkout 한다

    • 자식 브랜치를 만들고나서 파일을 수정하고 꼭 commit을 해야 가지가 쳐진다.

  • git flow feature finish Name : Name의 브랜치를 상위 브랜치와 merge하고 Name의 브랜치는 삭제 한다(remote와 local에서 같이 지운다)
