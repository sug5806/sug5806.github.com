2019-04-08 DataBase

2019/04/08 카테고리 데이터베이스

관계형 Data base

집합론에 근거한다.

  1. Oracle (대형 기업에서 가장 많이 사용함)
  2. MySQL (중소기업이 많이씀, PHP랑 정말 잘맞음 Ex) APM)
  3. Maria DB -> MySQL을 쓰던사람들이 많이 넘어옴
  4. postgreSQL -> 오픈 소스

DB에서의 세션을 열었다 = TCP connection을 했다

트랜잭션과 세션의 차이

DDL -> DLL은 자동으로 COMMIT이 반영된다.

  • craete
  • alter
  • drop
  • truncate -> 통째로 삭제해버리므로 delete보다 빠름

DML -> 자동으로 COMMIT이 되지않는다.

  • insert
  • select
  • update
  • delete -> 튜플을 한줄 한줄 삭제함

DCL

  • grant
  • revoke

TCL

  • commit
  • rollback

auto commit 이 0이라면
DDL은 자동으로 commit을 수반함

  • 객체
  • table(entity, relation)
    • 행 전체 : record, row ,tuple
    • 열의 처음 : field, attribute, colunm -> 자료형을 가지고 있어야 한다.
    • 어떤 속성은 중복값을 허용하지 않고 어떤 값은 허용하는 등의 조건 -> 제약조건(constraint)
  • view
  • index
  • schema
    • 내부 스키마
    • 외부 스키마
    • 물리 스키마

Maria DB

utf8 mb4

  • utf8이 한 글자당 최대 3byte 였으나 이모지가 나오면서 4byte로 확장함

@@ -> DB에서 사용하고있는 변수

SELECT USER, HOST FROM mysql.user;을 하면 모든 유저가 나옴
’%’ -> 내부, 외부모두 접속 허용
’localhost’ -> 로컬 호스트로 이용

GRANT ALL PRIVILEGES ON . TO ‘계정’@'localhost’
WITH GRANT OPTION; -> 모든 권한을 받고 다른사람에게 권한을 줄 수 있음

mysqldump -u sug5806 -p --databases my_db > mydb.sql -> sug5806 계정에 my_db라는 데이터베이스를 mydb.sql이라는 이름으로 덤프를 놓겠다

primary key를 지정하지 않아도 unique not null이 있다면 자동으로 이것을 주키로 지정한다

예시로 만들 TABLE

Student table

  • student ID -> INT, PRIMARY KEY
  • num_students -> INT NOT NULL
  • student_name -> VARCHAR, NOT NULL
  • height -> SMALLINT, DEFAULT 200
  • score -> SMALLINT, NULL
  • birthday -> DATE, NOT NULL
  • class ID -> INT, NULL,

Teachers table

  • teacher ID -> INT, PRIMARY KEY
  • subject -> VARCHAR
  • class ID -> INT, NULL

Classes table

  • class ID -> INT, PRIM_ARY KEY
  • num_student -> INT, NOT NULL개

SQL Query

예시

SELECT student_name, score FROM student_cp Where student_name like ‘Ma%’

  • student_cp라는 테이블에서 student_name과 score의 필드를 student_name 이 Ma로 시작하는 튜플을 가져와라

SELECT student_name, score FROM student_cp WHERE score > 50&& score < 70

  • student_cp라는 테이블에서 student_name과 score필드를 가져오는데 점수가 50점 초과 70점 미만을 가져와라
