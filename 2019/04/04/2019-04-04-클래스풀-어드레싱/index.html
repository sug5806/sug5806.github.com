Classful addressing

네트워크는 기본적으로 Network 와 Host로 나뉜다

IP주소를 체계적으로 다루기위해 도입한 개념 이다

그림을 보면 각 클래스를 구분하기위해 첫번째 바이트에 각 클래스마다 비트가 구별되어 있다.

  • A클래스는 0
  • B클래스는 10
  • C클래스는 110
  • D클래스는 잘 사용하지 않음
  • E 클래스의 경우 미래를 위해 남겨 둠

A 클래스 : 초/대규모의 네트워크에 할당.

  • 총 126개의 네트워크를 만들 수 있으며, 각 네트워크당 야 1677만개의 호스트를 연결 할 수 있음

    • 왜 126개 인가? 0.0.0.0(default route 또는 DHCP server를 탐지할 때 this host를 의미하는 source address로 사용하기도 함)과 127.0.0.0(loopback address)은 제외한다. 다른 클래스 역시 마찬가지이다.

    • P주소의 구성 : Network 주소. Host 주소. Host주소. Host주소

B 클래스 : 대규모 네트워크에 적용.

  • 총 16,382개의 네트워크를 만들 수 있으며 각 네트워크당 6만5천개의 노드를 연결 할 수 있다
  • IP주소의 구성 : Network 주소. Network주소. Host주소. Host주소

C 클래스 : 소규모 네트워크에 적용

  • 총 2,097,150개의 네트워크를 만들 수 있으며 각 네트워크마다 254개의 노드를 연결할 수 있다
  • IP주소의 구성 : Network 주소. Network주소. Network주소. Host주소

이러한 방식으로 IP를 할당하면 구분하기 힘드므로 subnetting이라는 개념을 도입하여 관리하기 수월하게 함
