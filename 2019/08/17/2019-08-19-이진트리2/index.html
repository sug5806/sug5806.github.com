개인적으로 공부한 것을 정리하였습니다.

이진 트리 운행(Traversals)

트리를 다루기 위해서는 트리를 운행하기 위한 메커니즘 필요한데 이것이 여기서 다룰 내용입니다. 트리의 모든 노드들은 방문하는 과정을 트리 운행이라 합니다. 각 노드들은 한 번만 처리되겠지만, 여러 번 방문을 받을 수 있습니다.

운행 방법(Traversal Possibilities)

운행 방법을 아래와 같이 3가지로 나타낼 수 있습니다.

  • 전위 운행법(Preorder)
  • 중위 운행법(Inorder)
  • 후위 운행법(Postorder)

여기에 순서를 따르지 않는 다른 운행법을 추가합니다.

  • 레벨 순서 운행법(Level Order Traversal): 이 함수는 너비 우선 탐색에서 영향을 받았습니다(그래프 알고리즘의 BFS).

전위 운행법(Preorder)

전위 운행법에서는 각 노드의 자신의 좌, 우측 하위 트리보다 먼저 처리됩니다.

  • root를 방문합니다.
  • 다시 전위 운행법으로 왼쪽 하위 트리를 운행합니다.
  • 전위 운행법으로 오른쪽 하위 트리를 운행합니다.

위의 트리에서는 1, 2, 4, 5, 3, 6, 7 순서로 처리됩니다.

중위 운행법(Inorder)

중위 운행법에서 root는 오른쪽 하위 트리와 왼쪽 하위 트리 사이에 방문하게 됩니다. 중위 운행법은 다음과 같이 정의됩니다.

  • 왼쪽 하위트리를 탐색합니다.
  • root를 처리합니다.
  • 오른쪽 하위 트리를 중위 운행법으로 탐색합니다.

위의 트리에서는 4, 2, 5, 1, 6, 3, 7 순서로 처리됩니다.

후위 운행법(Postorder)

후위 운행에서 root는 양쪽 하위 트리가 처리된 이후 처리됩니다. 후위 운행법은 다음과 같이 정의합니다.

  • 왼쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • 오른쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • root를 처리합니다.

앞의 예제 트리의 노드들은 4, 5, 2, 6, 7, 3, 1 순서로 처리됩니다.
