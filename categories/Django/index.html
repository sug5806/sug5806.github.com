views.py

POST 요청이 들어올시 if request.method == "POST"를 통하여 구분합니다. 그 다음 텍스트박스로 넘겨진 데이터를 request.POST.get('data')으로 받습니다.

  • 어떻게 name 데이터를 받았나요?

2019-06-14 URL 만들기

2019/06/14 카테고리 Django

URL 만들기

이번 시간에는 Django URL을 만드는 법을 알아보겠습니다

URLConf에 URL 패턴을 설정하고 이를 이용해 뷰가 동작되도록 설정하고 있습니다. 이 때 name 매개변수를 지정하면 해당 값으로 URL을 불러와 사용할 수 있습니다. 템플릿이나 뷰에서 URL을 하드코딩 한다면 URL 구조가 바뀌었을 경우 소스코드를 직접 수정해야 하는 불편함이 있기 때문에 동적으로 URL으 만들어주는 함수를 사용해야 합니다.

URL 만들기 함수들

- A. reverse
    가장 일반적으로 활용하는 메서드 입니다. URLConf에 있는 패턴 이름과 URL에 들어갈 매개변수를 전달합니다.



- B. reverse_lazy
    reverse의 lazy버전입니다. 클래스형 뷰의 success_url 등을 설정할 때 reverse를 사용하면 URLConf가 로드되기 전에 동작하기 때문에 오류가 발생합니다. 이런 경우
    reverse_lazy를 사용할 수 있습니다.



- C. resolve_url
    reverse의 간략화 버전입니다

2019-06-12 Django Shell 다루기2

2019/06/12 카테고리 Django

안녕하세요 이번시간에는 저번에 미처 다루지 못했던 QuerySet의 기능들을 알아보도록 하겠습니다.

미리 5개의 데이터를 만들어 놓았습니다.

  1. 전체 객체 불러오기

    • Model.objects.all()

  2. 특정 객체 불러오기

    • Model.objects.get(pk=값)
      • 일치한것만 가져옵니다. 일치한 것이 없을 시 오류를 발생합니다

        pk or id는 Django에서 DB 테이블을 작성하면 자동으로 추가시켜주는 필드입니다. 데이터를 추가할때마다 자동으로 1씩 증가합니다.

  3. 객체가 존재하는지 확인

    • Model.objetcs.filter(pk=1).exists()
      • 존재하면 True, 안하면 False를 반환합니다.

  4. 검색어로 검색하기

    • Model.objects.filter(필드명__옵션=값)
      • exact : 값 매치
      • iexact : 대소문자 구분없이 값 매치
      • startswith, istartswith : 값으로 시작하는 결과 찾기
      • endswith, iendswith : 값으로 끝나는 결과 찾기
      • contains, icontains : 값을 포함하는 결과 찾기
      • gt, gte : 값보다 큰경우, 크거나 같은 경우 찾기
      • lt, lte : 값보다 작은 경우, 작거나 같은 경우 찾기
      • 위의 옵션 말고도 여러가지 옵션이 있습니다.


필터를 연속해서 사용시 and 조건으로 연결됩니다.
이름이 자식이면서 나이가 10살 이상인 데이터

  1. 특정 조건 제외하기

    • Model.objects.exclude(조건)

      이름이 자식1을 제외한 데이터

  2. 갯수 끊어서 가져오기

    • Model.objects.all[:3] -> 3개만 가져오기

  3. 정렬하기

    • Model.objects.all().order_by(‘age’)

      나이를 기준으로 오름차순으로 정렬
      내림차순으로 정렬하고 싶다면 -를 붙여준다(-age)

  4. 조건의 or 연결

    • 필터는 기본적으로 and연산 입니다. or연산을 하고싶다면 Q객체를 사용해야 합니다.

      이름이 자식1 이거나 나이가 5살 초과인 데이터

  5. ForeignKey로 연결된 필드를 참조하는 경우

    • Model.objects.filter(parent__name=‘부모1’)

오늘은 admin page에서 수정과 link에 대해서 알아보겠습니다.

기존의 admin page입니다 여기서 나이나 주소를 수정하려면 일일히 들어가서 수정해야합니다. 매우 번거롭지 않나요? 이 문제를 해결하겠습니다.

admin.py를 수정합니다.

그런데 name을 변경하고 싶은데 할 수 없습니다. 왜냐하면, name에 link가 걸려있기 때문입니다. 이 문제를 해결하도록 하겠습니다.

그러나 이번엔 숫자가 너무 작아 클릭을 하기 힘듭니다. 이것을 수정하도록 하겠습니다
