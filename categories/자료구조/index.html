연결 리스트란 무엇인가요?

연결 리스트(Linked list)는 데이터 집합을 저장할 때 사용되는 자료 구조입니다. 연결 리스트는 다음과 같은 속성을 가집니다.

  • 연속된 요소들은 포인터로 연결됩니다.
  • 마지막 요소는 NULL을 가리킵니다.
  • 프로그램 실행 중에 크기가 늘거나 줄 수 있습니다.
  • 필요한 만큼 만들 수 있습니다.(시스템 메모리를 다 쓸 때까지).
  • 메모리 공간을 낭비하지 않습니다.
기본 구조

연결 리스트의 추상 데이터 형(ADT)

연결 리스트의 추상 함수는 다음과 같습니다.

  • 연결 리스트 주요 함수
    • Insert : 리스트에 요소를 추가합니다.
    • Delete : 리스트의 지정된 위치의 요소를 제거하고 반환합니다.
  • 연결 리스트의 보조 함수
    • Delete List : 리스트의 모든 요소들을 제거합니다.
    • Count : 리스트 내 모든 요소의 수를 반환합니다.

연결 리스트의 장점

연결 리스트는 장단점을 가지고 있습니다. 연결 리스트의 장점은 일정 시간(고정된 시간) 내에 확장될 수 있다는 것입니다. 배열의 경우 생성할 때 요소들의 수 만큼 메모리를 할당해야 합니다. 배열에 새로운 요소를 추가하기 위해서는 새로운 배열을 생성하고 기존의 배열의 요소들을 새로운 배열로 복사해야 하기 때문에 시간을 많이 소모 합니다. 연결리스트는 이러한 문제 없이 쉽게 새로운 요소를 추가할 수 있습니다.

연결 리스트의 단점

연결 리스트는 리스트 내 요소에 접근하는데 최악의 경우 O(n)을 가집니다. 비록 연결 리스트의 동적인 저장 공간의 할당이 큰 장점이라 하더라도 데이터의 정렬과 탐색의 오버헤드는 다른 상황을 만들 수 있습니다.

자료구조(Data Structure)

자료구조는 컴퓨터 내에서 유용하게 사용되도록 데이터를 저장하고 구조화하는 특정한 방법을 말합니다. 즉, 데이터를 저장하고 구조화하는 특별한 형태입니다. 대표적인 자료구조는 배열, 연결 리스트, 스택, 큐, 트리, 그래프 등이 있습니다.

자료구조는 구성요소들을 어떻게 구조화하였는가에 따라 두 가지 유형으로 분류됩니다.

  • 선형 자료구조(Linear data structures) : 선형 자료구조의 구성 요소들은 순차적으로 접근되지만, 모든 요소들이 반드시 순차적으로 저장되지는 않습니다. (Ex.연결 리스트, 스택, 큐)
  • 비선형 자료 구조(Non-linear data structures): 자료 구조의 구성 요소들이 비순차적인 순서로 저장되고 접근됩니다. (Ex. 트리, 그래프)

추상 자료형(Abstract Data Types - ADTs)

자료 구조와 연산을 결합하여 추상 자료형이라고 부릅니다. 추상 자료형은 두 부분으로 구성됩니다.

  • 데이터 선언
  • 연산자 선언
    일반적으로 사용되는 추상 자료형은 연결 리스트, 스택, 큐, 우선 순위 큐, 이진 트리, 딕셔너리, 분리 집합, 해시 테이블, 그래프 등이 있습니다. 예를 들어 스택은 자료 구조에 데이터를 저장할 때 LIFO(Last-In-First-Out) 메커니즘을 사용합니다. 스택에 마지막으로 삽입된 요소는 제일 먼저 빠져나와 제거됩니다. 스택의 일반적인 연산은 스택 생성, 스택에 삽입하는 push, 제거하는 pop, 최상위 데이터를 찾는 top 등이 있습니다.

2019-04-12 Tree

2019/04/12 카테고리 자료구조

Tree

트리는 그래프의 일종이다(Connected acyclic graph)

  • 그래프에서의 순회를 트리에서 적용할 수 있다

정의

  • 연결이 있고 사이클이 있는 그래프
  • 루트 노드(root)가 있다
  • 트리를 구성하는 노드 간에 단순 경로가 존재
    • 이동할때 갔던 경로를 다시 방문하지 않고 갈 수 있으면 단순경로가 있다고 한다
  • Start에서 출발해서 Start로 올수있다 (cyclic)
  • Connected : 경로가 무조건 하나는 있다

Tree 용어

  • 노드의 차수 : 자식의 갯수
  • 트리의 차수 : 노드의 차수 중에 가장 큰것
  • 리프 노드 : 자식이 없는 노드
  • 엣지 : 노드와 노드를 연결하는것
  • 트리의 높이 = 깊이 : 트리의 레벨
  • 트리가 되기위한 조건식 : 엣지 = 노드 - 1
  • 포레스트 : 서브 트리의 집합(Connected 만족 X) -> 트리가 모여있으므로 포레스트
  • 일반적으로 트리를 이야기 하면 이진트리(binary Tree)

Binary Tree

정의

  1. 어떤 노드의 최대 자식 수가 2개이하 인것
  2. 공집합 혹은 루트와 왼쪽 서브트리, 오른쪽 서브 트리로 이루어진 유한집합, 각각의 서브트리는 모두 이진 트리이다
    • 자식이 없지만 있다고 치는 노드를 공노드(Empty Node)라 한다.

특징

  • 각 Level에서의 최대 노드수 : 2^(level-1) 개
  • 트리 전체의 최대 노드 수 : 2^(Tree level) -1 개

포화 이진트리

  • 모든 레벨이 꽉 차 있다.

완전 이진트리

  • 일정 레벨 까지는 꽉 차있고 바로 밑 레벨부터 왼쪽에서 오른쪽으로 채워져있는 트리

편향 이진트리

  • 한쪽으로만 치우친 트리

트리의 순회

  • 재방문을 허용하지 않고 트리를 구성하는 모든 노드를 한번씩 방문하는것
순회의 계열은 2가지로 나뉜다

  • Stack계열

    • DFS(Depth First Search) - 그래프용어 : 깊이 우선 순회
      1. 전위순회(Preorder traversal) : A - B - C의 순서
      2. 중위순회(Inorder traversal) : B - A - C의 순서
      3. 후위순회(Post traversal) : B - C - A의 순서
    • 재귀 : 재귀를 하면 Stack Frame이 쌓임
    • 반복문 + Stack

  • Queue계열

    • BFS(Broath First Search) - 그래프용어: 너비 우선 순회
      • 트리에서는 레벨 순서 순회(level order)

BST(Binary Search Tree)

dictionary 구현하기

  1. hashing -> 파이썬의 dictionary는 hashing으로 구현됨
    • 테이블
  2. BST -> O(log N)
    • 이진트리의 일종
    • 모든 원소는 서로 다른 key를 가진다
    • 왼쪽 서브 트리에 있는 모든 키들은 루트의 키보다 작다.
    • 오른쪽 서브 트리에 있는 모든 키들은 루트의 키보다 크다.
    • 왼쪽 서브 트리와 오른쪽 서브 트리도 이진 탐색 트리이다.
    • 재귀적으로도 만족 해야한다 : 자식 트리역시 똑같은 로직으로 작동해야 한다

사족

  • delete() -> None : 리스트에서도 지워지고 메모리에서 지워짐
  • remove() -> removed data : 리스트에서 지워진 데이터를 반환(데이터를 반환 함으로써 사용자에게 어떻게 처리할건지 결정하게함)
  • python의 dictionary는 immutable 객체만 key로 사용할 수 있다
  • BST는 속도가 빠르기때문에 DB에서 사용하는 B-Tree역시 BST의 일종이다
  • BST는 Binary Tree의일종이므로 만들때는 상속으로 만드는게 좋다

리스트(List)와 배열(Array)의 차이점

  1. 동적배열(Dynamic Array) vs 연결리스트(Linked List)의 차이

    • DynamicArray

      • 장점:
        • search : O(1) -> Indexing을 사용하기 때문
          • 메모리에 가져올때 지역성이 적용되어 캐시히트가 일어나기 때문
      • 단점:
        • Insert : O(n) -> 삽입을 하기위해 기존의 n개의 데이터를 뒤로 밀어야 한다
        • Delete : O(n) -> 삭제를 한 뒤 남아있는 데이터를 앞으로 당겨야 한다.

    • LinkedList

      • 장점: 데이터의 삽입, 삭제가 매우 빠르다
        • Insert : O(1)
        • Delete : O(1)
      • 단점:
        • Search : O(n)
          • 지역성이 적용될 확률이 낮기때문에 캐시히트가 일어나기 힘들다

Search를 많이하면 배열을, Insert, Delete를 할때는 연결리스트를 주로 사용한다

다이나믹 힙(Dynamic Heap)기법으로 메모리 할당을 한쪽으로 몰아도 되도록이면 배열을 사용한다
