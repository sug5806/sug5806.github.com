2019-06-30 재귀 알고리즘이란?

2019/06/30 카테고리 알고리즘

개인적으로 공부한 것을 정리하였습니다.

재귀(Recursion)란?

자기 자신을 반복해서 호출하는 모든 함수를 재귀적(recursive)라 합니다. 재귀적 방법은 보다 작은 문제로 작업하기 위해서 자기 자신의 복사본을 호출함으로써 문제를 해결합니다. 이것을 재귀 단계라 합니다. 재귀 단계에서는 더 많은 동일한 재귀 호출이 발생할 수 있습니다. 재귀 호출은 종료되도록 하는 것이 중요합니다. 자기 자신을 호출할 때마다 원래 문제의 범위는 조금씩 줄어듭니다. 이 일련의 작은 문제들은 결국 base case에 도달합니다 [1]

왜 재귀방식을 사용할까?

일반적으로 재귀 코드는 반복 코드보다 짧고 쉽습니다. 재귀는 보다 작은 하위 작업으로 정의될 수 있는 작업에 가장 유용합니다. 예를 들어 정렬, 검색 그리고 탐색 문제는 간단한 재귀 방식을 가지고 해결합니다.

재귀함수의 예시

팩토리얼(factorial)을 예시로 들어보겠습니다.

위에서 설명한 것과같이 첫번째 함수를 호출 후 다음 함수를 호출할때는 value값이 1씩 줄어드는 것을 확인 할 수 있습니다.

  1. 재귀가 끝나는 조건을 말합니다 ↩

완전 검색 방법

  • 문제의 해법으로 생각할 수 있는 모든 경우의 수를 나열해보고 확인하는 기법, 다시말해 처음부터 끝까지 하나하나 대입해서 푸는 방법이다
    • Brute-force 혹은 Henerate-and-Test 기법이라고 불림
    • 모든 경우의 수를 테스트한 후, 최종 해법을 도출함
    • 일반적으로 경우의 수가 상대적으로 작을 때 유용함
    • 모든 경우의 수를 생성하고 테스트하기 때문에 수행 속도는 느리지만, 해답을 찾아내지 못할 확률이 적음
    • 주어진 문제를 풀 때, 우선 완전 검색으로 접근하여 해답을 도출한 후, 성능 개선을 위해 다른 알고리즘을 사용하고 해답을 확인하는 것이 바람직하다
  • Ex) Baby-gin 게임
      1. 6개의 숫자로 만들 수 있는 모든 숫자 나열(중복 포함)
      1. 앞의 3자리와 뒤의 3자리를 잘라, run과 triplete 여부를 테스트하고 최종적으로 Baby-gin을 판단함
  • Ex) 순열 : 서로 다른 것들 중 몇 개를 뽑아서 한 줄로 나열하는 것
    • 서로 다른 n개 중 r개를 택하는 순열은 nPr
    • nPr은 다음과 같은 식이 성립 : nPr = n*(n-1)(n-2) … * (n-r+1)
    • nPn = n! 이라고 표기하며 Factorial이라 부른다

2019-04-15 알고리즘이란?

2019/04/15 카테고리 알고리즘

알고리즘이란?

  • 유한한 단계를 통해 문제를 해결하기 위한 절차나 방법

알고리즘 표현방법

  • 슈도코드 : 특정 프로그래밍 언어의 문법을 따라 쓰여진 것이 아니라, 일반적인 언어로 코드를 흉내내어 알고리즘을 써 놓은 코드
    • 의사 코드로 흉내낸 코드
    • 실제적인 프로그래밍 언어로 작성된 코드처럼 컴퓨터에서 실행할 수 없음
    • 특정 언어로 프로그램을 작성하기 전에 알고리즘을 대략적으로 모델링하는데 쓰임
  • 순서도 : 프로그램이나 작업의 진행 흐름을 순서에 따라 여러 가지 기호나 문자로 나타낸 도표

무엇이 좋은 알고리즘인가?

  • 정확성 : 얼마나 정확하게 동작하는가?
  • 작업량 : 얼마나 적은 연산으로 원하는 결과를 얻어내는가?
  • 메모리 사용량 : 얼마나 적은 메모리를 사용하는가?
  • 단순성 : 얼마나 단순한가?
  • 최적성 : 더 이상 개선할 여지 없이 최적화 되어있는가?

알고리즘의 성능 분석

  • 많은 문제에서 알고리즘의 성능 분석기준으로 알고리즘의 작업량을 비교
    • Ex) 1~100까지의 합을 구하는 문제
    1. 실제 걸리는 시간을측정
    1. 실행되는 명령문의 개수를 계산 -> 일반적으로 많이 사용됨
    1. Big-O 표기법 : 시간 복잡도 함수 중에서 가장 큰 영향력을 주는 n에 대한 항만을 표시, 계수는 생략하여 표시
    • 요소 수가 증가함에 따라 각기 다른 시간 복잡도의 알고리즘은 아래와 같은 연산 수를 보임

2019-04-13 Greedy Algorithm

2019/04/13 카테고리 알고리즘

greedy algorithm

  • 각 단계마다 지금 당장 가장 좋은 방법만을 선택
  • 많은 경우 최적해를 찾지 못한다
  • 항상 최적해를 구할 수 있는 문제에 적용하고 수행 시간이 다른 알고리즘에 비해 훨씬 빠르다
  • 시간이나 공간적 제약으로 인해 다른 방법으로 최적해를 찾기 너무 어려운 경우 최적해 대신 근사해로 대체한다

예시

  • 동전 거스르기
    • 가장 큰 동전으로 거스르면 최적해이다
  • 회의실 배정
  • 최단경로

다익스트라 알고리즘

  • 아직 방문하지 않은 정점들 중 가장 거리가 짧은 정점을 방문
  • 정점에서 인접하고 아직 방문하지 않은 정점들의 거리 갱신
