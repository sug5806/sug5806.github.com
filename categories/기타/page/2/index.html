2019-05-31 Amazon CLI 설정하기

2019/05/31 카테고리 기타

Amazon CLI를 사용하여 사용자 등록하기

Django로 Amazon 서비스를 연동하다 보면 settings.py에 아마존 aws_access_key_id와 aws_secret_access_key를 입력해야 하는데 github 같은 곳에 실수로 올리게 되면 해커들이 도용을 하게됩니다. 이것 방지하기위해 AWS CLI를 통해 등록하면 settings.py에 더이상 access_key를 적지 않아도 됩니다.

  1. 먼저 아마존에 가입을하고 IAM항목으로 들어갑니다.

  2. 그룹을 누르고 새로운 그룹생성을 합니다.

  3. 그룹이름을 적고 우측하단에 다음단계를 누릅니다

  1. 본인이 이용하실 서비스에 대한 정책을 선택하고 우측 하단에 다음단계로 진행합니다. 저는 S3를 이용할것이므로 S3FullAccess를 선택하였습니다.
  1. 선택하신것을 확인하시고 우측 하단에 그룹선택을 눌러줍니다.
  1. 이제 그룹에 사용자를 추가시켜줘야 합니다. 사용자 탭을눌러 사용자 추가를 해줍니다.
  1. 사용자 이름을 적어주시고 우리는 CLI를 통해 등록을 할것이므로 프로그래밍 방식 액세스에 체크하도록 합니다. 그리고 우측 하단의 다음으로 넘어갑니다
  1. 방금 만든 그룹이 보이실텐데 추가하고 싶은 그룹을 체크박스 하신뒤 우측 하단의 다음버튼을 눌러줍니다
  1. 태그 추가는 지금당장 할일이 없으므로 생략하도록 합니다
  1. 이제 그동안 설정한 값들을 확인할 수 있는 화면입니다. 확인후 사용자 만들기를 누릅니다.
  1. 제일 중요한 화면입니다. 이 화면을 벗어나게되면 확인할 수가 없습니다. 그러므로 .csv 다운로드를 통해 등록한 사용자의 ID와 시크릿키를 다운받습니다. 절대 다른사람에게 노출이 되면 안됩니다.
  1. 이제 django에서 사용자를 등록하도록 하겠습니다.
    pip install awscli를 입력하여 설치해줍니다
  1. 설치가 다 되면 aws configure을 통해 다운받으신 csv파일의 ID와 Key를 입력하시면 됩니다. region은 서울이니 ap-northeast-2, output format은 json, text, table중 하나를 입력해주면 됩니다.
  1. 등록을 마치면 더이상 settings.py에 ID와 Key를 입력하지 않아도 자동으로 본인 root디렉토리에 .aws디렉토리 내부의 파일을 읽어 사용할 수 있습니다.

2019-05-19 아마존 인증서 발급받기

2019/05/19 카테고리 기타

아마존 인증서 발급 받기

Domain

  • naver.com

sub Domain

  • blog.naver.com
  • static.web.nvaer.com

이메일 검증이 편하긴 하다(1년마다 갱신 해야함)
DNS검증을 하게되면 자동으로 갱신이 된다.

Certificate Manager로 들어간다

[인증서 요청]을 클릭하게되면 위와 같은 화면이 나온다
*(와일드카드)로 발급받게되면 서브도메인 전체에 대한 적용이 가능하다

검증방법에서는 DNS 검증을 선택하고 [검토]버튼을 누른다

검증 방법과 신청 내용을 확인하고 [확인 및 요청]버튼을 클릭한다

도메인 옆의 삼각형을 누르면 상세적보가 나오는데 [Route 53에서 레코드 생성] 버튼을 클릭한다
레코드 내용을 확인하고 [생성] 버튼을 클릭한다

생성 성공 메시지를 확인하고 [계속]버튼을 클릭한다

다시 인증서 목록으로 돌아오면 검증 미완료 상태를 확인할 수 있고
시간이 지나면 검증 완료가 되고 인증서가 발급된다

2019-05-01 웹프레임워크와 라이브러리의 차이

2019/05/01 카테고리 기타

프레임워크란?

소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔 일련의 협업화된 형태로 클래스들을 제공하는 것

프레임워크는 라이브러리와 달리 애플리케이션의 틀과 구조를 결정할 뿐 아니라, 그 위에 개발된 개발자의 코드를 제어한다.

  • 프로그램 기본구조(뼈대) -> 자동차 프레임

라이브러리란?

자주 쓰일 만한 기능들을 모아놓은 유틸(클래스)들의 모음집

  • 자동차 바퀴, 자동차 헤드라이트, 자동차 에어백

그럼 둘의 차이는 뭘까?

라이브러리는 도구(연장)의 개념이다

  • 망치, 톱, 삽 등등
    프레임워크는 탈것의 개념이다
  • 비행기, 차, 배 등등

사람은 도구를 선택하는 입장이기 때문에, 어떤 도구를 사용하든 원하는 것을 만들어낼 수 있으면 된다.

반면에 탈것은 정해진 곳으로만 다녀야 한다. 그 목적에 맞게 만들어져 있기 때문에 정해진 규칙에 맞춰 엔진, 기어, 핸들만 잘 다루면 된다.

라이브러리와 달리 프레임워크는 이미 프로그래밍할 규칙이 정해져있다.
