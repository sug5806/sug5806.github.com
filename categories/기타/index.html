Ajax란?

Ajax는 JavaScript의 라이브러리중 하나이며 Asynchronous Javascript And Xml(비동기식 자바스크립트와 xml)의 약자입니다. 브라우저가 가지고있는 XMLHttpRequest 객체를 이용해서 전체 페이지를 새로 고치지 않고도 페이지의 일부만을 위한 데이터를 로드하는 기법이며 Ajax를 한마디로 정의하자면 JavaScript를 사용한 비동기 통신, 클라이언트와 서버간에 XML데이터를 주고받는 기술이라고 할 수 있습니다.

출처: 코딩팩토리

실생활에서 사용하는 앱중에서 예를들면 인스타그램이나 페이스북의 좋아요를 눌렀을시 새로고침 없이 작동하거나 무한스크롤 기능이 있습니다.

다음 시간에는 Django에 Ajax를 적용하도록 하겠습니다.

2019-05-31 Amazon CLI 설정하기

2019/05/31 카테고리 기타

Amazon CLI를 사용하여 사용자 등록하기

Django로 Amazon 서비스를 연동하다 보면 settings.py에 아마존 aws_access_key_id와 aws_secret_access_key를 입력해야 하는데 github 같은 곳에 실수로 올리게 되면 해커들이 도용을 하게됩니다. 이것 방지하기위해 AWS CLI를 통해 등록하면 settings.py에 더이상 access_key를 적지 않아도 됩니다.

  1. 먼저 아마존에 가입을하고 IAM항목으로 들어갑니다.

  2. 그룹을 누르고 새로운 그룹생성을 합니다.

  3. 그룹이름을 적고 우측하단에 다음단계를 누릅니다

  1. 본인이 이용하실 서비스에 대한 정책을 선택하고 우측 하단에 다음단계로 진행합니다. 저는 S3를 이용할것이므로 S3FullAccess를 선택하였습니다.
  1. 선택하신것을 확인하시고 우측 하단에 그룹선택을 눌러줍니다.
  1. 이제 그룹에 사용자를 추가시켜줘야 합니다. 사용자 탭을눌러 사용자 추가를 해줍니다.
  1. 사용자 이름을 적어주시고 우리는 CLI를 통해 등록을 할것이므로 프로그래밍 방식 액세스에 체크하도록 합니다. 그리고 우측 하단의 다음으로 넘어갑니다
  1. 방금 만든 그룹이 보이실텐데 추가하고 싶은 그룹을 체크박스 하신뒤 우측 하단의 다음버튼을 눌러줍니다
  1. 태그 추가는 지금당장 할일이 없으므로 생략하도록 합니다
  1. 이제 그동안 설정한 값들을 확인할 수 있는 화면입니다. 확인후 사용자 만들기를 누릅니다.
  1. 제일 중요한 화면입니다. 이 화면을 벗어나게되면 확인할 수가 없습니다. 그러므로 .csv 다운로드를 통해 등록한 사용자의 ID와 시크릿키를 다운받습니다. 절대 다른사람에게 노출이 되면 안됩니다.
  1. 이제 django에서 사용자를 등록하도록 하겠습니다.
    pip install awscli를 입력하여 설치해줍니다
  1. 설치가 다 되면 aws configure을 통해 다운받으신 csv파일의 ID와 Key를 입력하시면 됩니다. region은 서울이니 ap-northeast-2, output format은 json, text, table중 하나를 입력해주면 됩니다.
  1. 등록을 마치면 더이상 settings.py에 ID와 Key를 입력하지 않아도 자동으로 본인 root디렉토리에 .aws디렉토리 내부의 파일을 읽어 사용할 수 있습니다.

2019-05-01 웹프레임워크와 라이브러리의 차이

2019/05/01 카테고리 기타

프레임워크란?

소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔 일련의 협업화된 형태로 클래스들을 제공하는 것

프레임워크는 라이브러리와 달리 애플리케이션의 틀과 구조를 결정할 뿐 아니라, 그 위에 개발된 개발자의 코드를 제어한다.

  • 프로그램 기본구조(뼈대) -> 자동차 프레임

라이브러리란?

자주 쓰일 만한 기능들을 모아놓은 유틸(클래스)들의 모음집

  • 자동차 바퀴, 자동차 헤드라이트, 자동차 에어백

그럼 둘의 차이는 뭘까?

라이브러리는 도구(연장)의 개념이다

  • 망치, 톱, 삽 등등
    프레임워크는 탈것의 개념이다
  • 비행기, 차, 배 등등

사람은 도구를 선택하는 입장이기 때문에, 어떤 도구를 사용하든 원하는 것을 만들어낼 수 있으면 된다.

반면에 탈것은 정해진 곳으로만 다녀야 한다. 그 목적에 맞게 만들어져 있기 때문에 정해진 규칙에 맞춰 엔진, 기어, 핸들만 잘 다루면 된다.

라이브러리와 달리 프레임워크는 이미 프로그래밍할 규칙이 정해져있다.

2019-04-10 Git 특강2 branch

2019/04/10 카테고리 기타

Git

브랜치란?

  • 분기점을 생성하고 독립적으로 코드를 변경할 수 있도록 도와주는 모델

  • Ex)

    • matser branch

    • another branch

  • 로컬의 branch와 리모트의 branch는 따로 존재한다

  • git branch -r : 리모트의 브랜치를 조회한다

  • git branch : 로컬의 브랜치를 조회한다

브랜치 생성

  • git branch feat/loop -> feat/loop라는 새로운 브랜치를 만든다
    • 현재 브랜치의 현재 상태 그대로 복제하여 만든다

브랜치 이동

  • git checkout feat/loop -> feat/loop 브랜치로 이동한다.

리모트 레포와 동기화

  • git push -u origin feat/loop : 리모트 레포에 feat/loop라는 새로운 브랜치 생성
    • -u : upstream의 약자 -> 리모트 레포에 feat/loop가 없으므로 해주어야 한다.

브랜치를 해도 deletion이 없으면 평행으로 되고 있으면 밑으로 추가되고, 우리가 논리적으로 생각하는 것으로 그림을 그린다.

브랜치 병합하기

  1. 우선 병합하고자 하는 브랜치로 이동한다 -> git checkout feat/loop
  2. git merge feat/con-state : feat/loop를 feat/con-state의 내용과 동기화시킨다(feat/con-state와 같아진다)
  3. 자동으로 commit이 생긴다.

브랜치 삭제하기

  • git branch -D/d 브랜치명
    • D는 강제로 삭제한다

브랜치및 머지를 보다 쉽게 하기

  • git flow 전략 : 다운로드 링크

    • master는 항상 개발이 끝난 것만 보여야한다

  • git init -> git flow init : git init을 하면서 flow까지 한꺼번에 함

  • develop 브랜치를 만들어도 기능 개발을 할때는 브랜치를 또 만들어서 해야한다

  • git flow feature start Name : Name의 브랜치를 생성하고 checkout 한다

    • 자식 브랜치를 만들고나서 파일을 수정하고 꼭 commit을 해야 가지가 쳐진다.

  • git flow feature finish Name : Name의 브랜치를 상위 브랜치와 merge하고 Name의 브랜치는 삭제 한다(remote와 local에서 같이 지운다)

정규 표현식

  • r`[] -> row string(파이썬에서만 쓰임)

Match -> 문자열의 처음부터 찾고 안맞으면 False
Search -> 문자열 중에서 하나라도 있으면 return

  • 반복

    1. * : 0 ~ 무한대 -> {0, 무한대}
    2. + : 1 ~ 무한대 -> {1, 무한대}
    3. ? : 0 ~ 1 -> {0, 1}
    4. {m} : m개
    5. {m,n} : m개부터 n개까지

  • ^

    1. 일반적인 경우 문자열의 시작
    2. [^abc] : a,b,c가 모두 아닌 모든 문자열

  • $

    1. 일반적인 경우 문자열의 끝

  • .

    • 일반적으로는 모든 문자들 나타냄

  • r

    • r’ '안에 있으면 모든 특수문자들이 효력을 잃음
  • 특수문자
    • \ : \
    • \d : [0-9]
    • \D : [^0-9]
    • \s(white space) : \t, \n, space, \r
    • \S : not white space
    • \w : [a-zA-Z0-9]
    • \W : [^a-zA-Z0-9]
  • Group
    • pattern = r’([0-9]{3})[-./ ](\d{3,4})[-./ ](\d{4})’
    • 그룹을 3개로 나눔

Ex) 변수할당 구분하기

  • r’([a-zA-Z_]\w*)\s*=\s*([0-9]+)’

Init으로 시작하는 방법

레포 이름과 폴더 이름이랑 같지 않으면 나중에 헷갈릴수 있으므로 똑같이 하는게 좋다

  1. 본인의 git 저장소로 쓰고싶은 로컬 디렉토리로 이동 한다음 git init을 한다

  2. git config --global user.name "username"을 입력한다

  3. git config --global user.email "email"을 입력한다

    • git 설정을 할때 git config --global을 하면 전체에 설정이 되고 global을 빼면 해당 레포?에서만 지정이 된다.

  4. git remote

    • get-url 별명 : 별명이 가지고 있는 주소를 알 수 있다
    • 별명 주소 : 별명으로 주소의 리모트 레포를 이용 하겠다 반드시 별명이 있어야 함

  5. git add

    • git add 파일이름을 하여 staing으로 올려준다.

  6. commit을 할때는 동작을 하는 코드만 commit을 해야한다

    • -u를 붙임으로써 같은 브랜치라고 알려준다(브랜치 당 최초 1번만 하면 된다)
      • (git commit -u 별명 master)
    • commit을 할때 기능에 따른 말머리를 추가하는것이 좋다
    • commit 메시지를 작성할때는 현재형으로 작성하는것이 좋다
    • commit 메시지를 적을때는 제목으로 무엇을 변경했는지 알아볼수 있게 작성하는게 좋다
    • collection (list, dictionary, tuple 등)을 사용할때는 나열식으로 하는것 보다는 값마다 한줄 한줄씩 하는게 좋다
    • collection을 사용할때는 마지막 원소에 ,(콤마)를 적어주면 깃에서 추가된 내용만 추가가 된다.
    • commit을 할때는 동작하는 최소 단위로 해주는 것이 좋다(function단위도 좋은 방법)

2019/03/22 카테고리 기타

Procedural vs Object-Oriented

  • procedural -> 함수를 이용해서 추상화를 했다
  • Object-Oriented -> 객체를 통해 추상화를 했다
    • 객체 단위로 프로그래밍을 한다
    • 객체(Object) -> ‘관련있는’ 변수(상태정보)와 ‘관련있는’ 함수를 한곳에 모아놓은 ‘곳’ (곳 = 메모리) -> bundling
      • attribute(속성)
        • instance member(변수)
          • 각각의 객체가 가지는 값
        • instance method(함수)
          • 객체가 할 수 있는 행동, 기능

message passing

  • 다른 객체(other)의 상태정보(멤버)를 변경할 때는 반드시 상대 객체가 가진 메서드를 이용한다

클래스는 함수의 모임

객체와 인스턴스의 차이
객체와 인스턴스의 메모리는 같음
인스턴스=> Ex) me라는 객체는 Account라는 클래스의 인스턴스

OOP

  1. 캡슐화(Encapsulation)

    1. 관련있는 멤버(변수) 와 메서드(함수)를 하나의 단위(대부분 클래스)로 묶는 것
      1. 클래스를 제외하고도 하나의 단위로 묶을수 있다면 OOP라고 함
        1. Ex) C에서의 구조체
        2. the bundling of data with methods
      2. 정보 은닉을 포함함

  2. 정보은닉( 파이썬은 완벽한 정보은닉을 지원 안함) -> C++에서 클래스 만들면서 private선언

    1. 어떤 멤버를 공개 / 비공개
    2. 어떤 메서드를 공개 / 비공개

  3. 다형성

    1. 상속
      • 상속을 쓰지 말자
      • 상속은 OOP의 정수이다
        • 상속은 쓰되 다중상속은 하지말자
      • 멤버변수에는 절대 직접접근 금지
      • 객체의 멤버에 접근 -> getter / 수정 -> setter 을 수행할때는 method function을 이용해야함
      • 내부에서는 멤버처럼 쓰는것처럼 하면서 property를 사용하여 정보은닉을 함
      • 인스턴스 멤버를 만든다고하면 지금 당장 쓰지 않아도 되도록이면 생성자에서 만들어 주는게 좋음
      • 멤버는 어디에서든지 생성될 수 있음
    2. Relations between classes
      • choose either inheritance or composition (언제 상속으로하고 언제 컴포지션으로 하는가)
        • IS - A
          • A laptop IS - A computer / ~은 ~의 한 종류이다
          • Inheritance (상속)
          • 조건 1 : 모든 멤버와 모든 메서드의 중복이 되어야함
          • 재사용성(reusability) -> 재사용성 만으로 IS -A 상속을 하면 절대 안되고 조건을 모두 충족해야함!
        • HAS - A
          • A policeman has a gun / ~이 ~을 가지고 있다. or 포함하고 있다
          • Composition(합성) -> ◆(In UML)
            • same life cycle = 강하게 연결되어 있다
          • Aggregation(통합) -> ◇(In UML)
            • NOT same life cycle = weak coupling
    3. method overriding (메소드 오버라이딩) -> 상속이 필수
      • polymophism (다형성)
        • 상속 관계에 있는 다양한 클래스의 객체에서 같은 이름의 메서드를 호출할 때, 각 객체가 서로 다르게 구현된 메서드를 호출함으로써 서로 다른 행동, 기능, 결과를 가져오는 것
    4. abstract class (추상 클래스) = 자바에서의 인터페이스
      • 인스턴스를 만들 수 없다.
      • 하나 이상의 abstract method (추상 메소드)
        • 몸체(body)가 없는 함수 -> pass
      • 파생 클래스에서 반드시 overriding을 해야함 -> 구현을 강제하도록 한다

문자와 문자열, 함수전달 방식

  1. 문자, 문자열 -> character set, ASCII( -> UNI-CODE(테이블로 이루어져 있음)-> UTF-8, UTF-16, UTF-32)

    • character set -> 문자 집합(Ex. 라틴문자)

    • character encoding -> 문자 인코딩(Ex. 모스 부호)

    • code point -> 문자 하나에 정수 하나를 매핑해둠(Ex. 아스키 코드표(7bit로 모두 표현가능)

    • UniCode -> 2byte 기반으로 만듬(2byte 테이블을 17개 만들어놓았고 현재 실생활에서 쓰는건 2개(4개 등록되어있음))

      • 216개만큼의 문자가 매핑되어있음 BMP -> BMP까지는 2byte면 다 표현이 가능
      • 아스키는 코드포인트 자체 그대로 메모리에 저장함
      • 유니코드는 메모리에 저장할때 utf-8, utf-16, utf-32로 나눠서 저장
      • UTF-16의 경우 BMP에 해당하는 문자는 그대로 전송(2byte), 2차평면 이상부터는 4byte로보내는 대신 앞을 0으로 다 채워서 보냄
      • UTF-32는 처음부터 4byte로 바꿔서 보냄
      • UTF-8 -> 가변길이 인코딩, 한글은 3byte로 인식, 엔디언 변환을 무시(Endianness)하고 그대로 저장, 전송을 함

    • 사람들이 인식하는 것은 빅엔디언형식이고 컴퓨터는 대부분 Intel CPU를 사용하고 intel은 리틀엔디언 방식으로 저장함

U+AC00 을 메모리에 저장할때는 리틀엔디언 방식을 통해 00 AC로 저장함
네트워크를 통해 데이터를 보낼때는 빅엔디언으로 보냄
실제로 네트워크를 통해 전달되는 방식으로 변환되는것이 유니코드인코딩방식?

  1. 함수를 호출할때 인자 전달 방식 ->> call by value, call by reference -> C언어 call by object reference(call by assignment) -> 파이썬
    • call by value : 값에 의한 전달 -> 함수 스택프레임 안에서 값이 바뀌어도 밖의 스택프레임에는 영향을 주지않는것을 보장함
    • call by reference : 주소에 의한 전달 -> 함수 스택프레임 안에서 주소를 통해 바깥 스택프레임에 있는 변수에 영향을주어 값을 변화시킴
