2019-05-08-Class-Base-View(CBV)란?

2019/05/08 카테고리 Django

Class Base View(CBV)

클래스형 뷰는 상속과 믹스인 기능을 이용하여 코드를 재사용하고 뷰를 체계적으로 구성할 수 있다

클래스형 뷰의 장점

  • GET, POST등 HTTP 메소드에 따른 처리 코드를 작성할 때 if 함수 대신에 메소드 명으로 코드의 구조가 깔끔하다.
  • 다중상속 같은 객체지향 기법을 활용해 제너릭 뷰, 믹스인 클래스 등을 사용해 코드의 재사용과 개발 생산성을 높여준다.

제너릭 뷰와 상속

제네릭 뷰

뷰 개발 과정에서 공통적으로 사용할 수 있는 기능들을 추상화하고, 장고에서 기본적으로 제공해주는 클래스형 뷰

제너릭 뷰의 4가지 분류

  • 기반뷰(Base View): 뷰 클래스를 생성하고, 다른 제너릭 뷰의 부모 클래스가 되는 기본 제너릭 뷰
  • 제너릭 보기 뷰(Generic Display View) : 객체의 목록 또는 하나의 객체 상세 정보를 보여주는 뷰
  • 제너릭 수정 뷰(Generic Edit View): 폼을 통해 객체를 생성, 수정, 삭제하는 기능을 제공하는 뷰
  • 제너릭 날짜 뷰(Generic Date View): 날짜 기반 객체의 연/월/일 페이지로 구분해 보여주는 뷰

주요 제네릭 뷰 목록

  1. 기반 뷰(Base View)

    • View: 최상위 부모 제너릭 뷰 클래스
    • TemplateView: 주어진 템플릿으로 렌더링
    • RedirectView: 주어진 URL로 리다이렉트

  2. 제너릭 보기 뷰(Generic Display View)

    • DetailView: 조건에 맞는 하나의 객체 출력
    • ListView: 조건에 맞는 객체 목록 출력

  3. 제너릭 수정 뷰(Generic Edit View)

    • FormView: 폼이 주어지면 해당 폼을 출력
    • CreateView: 객체를 생성하는 폼 출력
    • UpdateView: 기존 객체를 수정하는 폼을 출력
    • DeleteView: 기존 객체를 삭제하는 폼을 출력

  4. 제네릭 날짜뷰(Generic Date View)

    • YearArchiveView: 주어진 연도에 해당하는 객체 출력
    • MonthArchiveView: 주어진 월에 해당하는 객체 출력
    • DayArchiveView: 주어진 날짜에 해당하는 객체 출력
    • TodayArchiveView: 오늘 날짜에 해당하는 객체 출력
    • DateDetailView: 주어진 연, 월, 일 PK(또는 Slug)에 해당하는 객체 출력

속성 변수 오버라이딩

  1. model : 기본 뷰(View, Template, RedirectView) 3개를 제외한 모든 제너릭 뷰에서 사용한다.
  2. 기본 뷰(View, Template, RedirectView) 3개를 제외한 모든 제너릭 뷰에서 사용한다. queryset을 사용하면 model 속성은 무시된다.
  3. template_name : TemplateView를 포함한 모든 제너릭 뷰에서 사용한다. 템플릿 파일명을 문자열로 지정한다.
  4. success_url : FormView, CreateView, UpdateView, DeleteView에서 폼에 대한 처리가 성공한 후 리다이렉트할 URL 주소이다

urls에서 연결하기

  1. as_view() 메소드에서 클래스의 인스턴스를 생성한다.
  2. 생성된 인스턴스의 dispatch()메소드를 호출한다.
  3. dispatch() 메소드는 요청을 검사해서 HTTP의 메소드(GET, POST)를 알아낸다.
  4. 인스턴스 내에 해당 이름을 갖는 메소드로 요청을 중계한다
  5. 해당 메소드가 정의되어 있지 않으면, HttpResponseNotAllowd 예외를 발생시킨다

클래스 형 뷰에서의 폼 처리

클래스형 뷰에서의 폼처리는 get, post 방식을 메소드로 구분하여 처리함으로써, 코드구조가 깔끔해진다.

이 코드를 위에서 봤었던 제네릭 뷰인 FormView 를 상속받아 처리하면 아래와 같이 더 간결해지는 것을 볼 수 있다.

  • form_class: 사용자에 보여줄 폼을 정의한 forms.py 파일 내의 클래스명
  • template_name: 폼을 포함하여 렌더링할 템플릿 파일 이름
  • success_url: MyFormView 처리가 정상적으로 완료되었을 때 리다이렉트 될 URL
  • form_valid() 함수: 유효한 폼 데이터로 처리할 로직 코딩. 반드시 super() 함수를 호출해야 함.

2019-05-01 웹프레임워크와 라이브러리의 차이

2019/05/01 카테고리 기타

프레임워크란?

소프트웨어의 구체적인 부분에 해당하는 설계와 구현을 재사용이 가능하게끔 일련의 협업화된 형태로 클래스들을 제공하는 것

프레임워크는 라이브러리와 달리 애플리케이션의 틀과 구조를 결정할 뿐 아니라, 그 위에 개발된 개발자의 코드를 제어한다.

  • 프로그램 기본구조(뼈대) -> 자동차 프레임

라이브러리란?

자주 쓰일 만한 기능들을 모아놓은 유틸(클래스)들의 모음집

  • 자동차 바퀴, 자동차 헤드라이트, 자동차 에어백

그럼 둘의 차이는 뭘까?

라이브러리는 도구(연장)의 개념이다

  • 망치, 톱, 삽 등등
    프레임워크는 탈것의 개념이다
  • 비행기, 차, 배 등등

사람은 도구를 선택하는 입장이기 때문에, 어떤 도구를 사용하든 원하는 것을 만들어낼 수 있으면 된다.

반면에 탈것은 정해진 곳으로만 다녀야 한다. 그 목적에 맞게 만들어져 있기 때문에 정해진 규칙에 맞춰 엔진, 기어, 핸들만 잘 다루면 된다.

라이브러리와 달리 프레임워크는 이미 프로그래밍할 규칙이 정해져있다.

2019-04-25 데이터 베이스 기초

2019/04/25 카테고리 데이터베이스

데이터 베이스 기초

데이터 베이스 4가지 정의 : 통합된 데이터, 운영 데이터, 공용 데이터, 저장된 데이터

데이터 베이스의 특징 : 실시간 접근성, 내용에 의한 참조, 동시 공용, 계속적인 변화

데이터 베이스 시스템 : 데이터베이스를 이용하여 자료를 저장하고 관리하여 정보를 얻어내는 데 필요한 컴퓨터 중심의 시스템

분산 데이터베이스 : 물리적으로는 컴퓨터 네트워크를 통해 분산되어 있는 데이터 베이스

데이터 웨어하우스 : 급증하는 다량의 데이터를 효과적으로 분석하여 정보화하고 이를 여러 계층의 사용자들이 효율적으로 사용 할 수 있도록 한 데이터베이스

DBMS : 사용자와 데이터베이스 사이에서 사용자의 요구에 따라 정보를 생성해 주고, 데이터베이스를 관리해 주는 소프트웨어. 기존에 있던 데이터 중복성과 데이터 종속성 해결

DBMS의 필수기능 : 정의, 조작, 제어

논리적 독립성 : 논리적 구조를 변경시켜도 응용프로그램에 영향 없음

물리적 독립성 : 물리적 구조를 변경시켜도 응용프로그램에 영향 없음

데이터 사전(= 시스템 카탈로그) : 데이터베이스에 저장되어 있는 모든 데이터 개체들에 대한 정보를 유지,관리하는 시스템. 카탈로그에 저장된 정보를 메타 데이터라고 함. 좁은의미로 자료 사전이라 함.
스키마는 컴파일되어 데이터 사전에 저장됨.

인스턴스(Instance) : 속성이 구체적인 데이터 값을 가지고 있는것

외부스키마 = 서브 스키마 = 사용자 뷰 : 사용자나 응용 프로그래머가 각 개인의 입장에서 필요로 하는 데이터베이스의 논리적 구조를 정의한 것 뷰 정의를 허용하는 단계

개념 스키마 : 단순히 스키마라 하면 개념 스키마를 의미, 기관이나 조직체의 관점에서 데이터베이스를 정의

내부 스키마 : 데이터베이스의 물리적 구조, 데이터의 실제 저장방법을 기술

DDL(데이터 정의 언어) : 번역한 결과가 데이터 사전에 저장됨

DML(데이터 조작 언어) : 사용자로 하여금 데이터를 처리할 수 있게 하는 도구로서, 사용자와 DBMS간의 인터페이스를 제공함. 포트란, 코볼등의 호스트 언어에서 DB기능을 추가해서 만든 언어

DCL(데이터 제어 언어) : 데이터에 대한 액세스(권한)을 제어하기위한 데이터베이스 언어

질의어 : 단말 사용자가 쉽게 DB를 액세스 할 수 있도록 대화식의 자연어로 만든 비 절차적 조작언어

요구조건분석 : 데이터베이스를 사용할 사람들로부터 필요한 용도를 파악함

개념적 설계 : 현실 세계에 대한 인식을 추상적 개념으로 표현하는 과정 (개념 스키마 모델링과 트랜잭션 모델링을 병행수행)

논리적 설계 : 컴퓨터가 처리할 수 있는 물리적 저장장치에 저장할수 있도록 변환하기 위해 특정 DBMS가 지원하는 논리적 자료 구조로 변환시키는 과정

물리적 설계 : 논리적 설계 단계에서 논리적 구조로 표현된 데이터를 디스크 등의 물리적 저장 장치에 저장할 수 있는 물리적 구조의 데이터로 변환하는 과정.

데이터 베이스 구현 : 논리적 설계 단계와 물리적 설계 단계에서 도출된 데이터베이스 스키마를 파일로 생성하는 단계

데이터 모델 : 현실 세계의 정보들을 컴퓨터에 표현하기 위해서 단순화, 추상화하여 체계적으로 표현한 개념적 모형
개념적 데이터 모델 : 인간의 이해를 돕기위해 현실세계에 대한 인식을 추상적으로 표현하는 과정. 정보 모델이라고도 함. ex) E-R모델

논리적 데이터 모델 : 컴퓨터 세계의 환경에 맞도록 변환하는 과정. 단순히 데이터 모델이라 하면 논리적 데이터 모델을 의미

논리적 데이터 모델 종류

  1. 계층형 데이터 모델 : 트리구조를 이용하여 표현. 상위와 하위가 일 대 다의 대응관계 개체 = 세그먼트 : 트리를 구성하는 노드 정보가 전달되는 단위. 개체간의 관계를 부모 자식의 관계로 표현
  2. 망형 데이터 모델 : 그래프 구조. 상위와 하위가 M : N관계. 개체간 관계를 오너와 멤버의 관계로 표현
  3. 객체 지향형 데이터모델(OODM) : 객체 개념을 데이터베이스에 도입함.
  4. 객체 지향형 데이터 베이스(OODB) : 객체지향 데이터 모델의 개념을 반영하여 실세계에 존재하는 객체를 표현하고 관리하는 데이터 베이스 기술
  5. 관계형 데이터 모델 : 표(table)을 이용하여 데이터 상호관계를 정의하는 DB구조

튜플의 수 : 카디널리티, 기수, 대응수

속성의 수 : 디그리(degree), 차수

도메인 : 하나의 애트리뷰트가 위하라 수 있는 같은 타입의 원자값들의 집합

후보키 : 기본키로 사용할수 있는 속성들. 유일성, 최소성

대체키 : 기본키를 제외한 나머지 후보키

슈퍼키 : 속성들의 집합으로 구성된 키. 유일성 만족 최소성 X

무결성

무결성 : 데이터 베이스에 저장된 데이터 값과 그것이 표현하는 현실 세계의 실제값이 일치하는 정확성
널 무결성 : 특정 속성값이 NULL이 될 수 없음

고유(Unique) 무결성 : 각 튜플이 갖는 값들이 서로 달라야 한다는 규정

도메인 무결성 : 특정 속성의 값이, 속성이 정의된 도메인에 속한 값이 되어야 한다는 규정

키 무결성 : 하나의 테이블에는 적어도 하나의 키가 존재해야함

관계 무결성 : 관계에 대한 적절서 여부 지정

개체 무결성 : 릴레이션에서 기본키를 구성하는 속성은 널값이나 중복값을 가질수 없음

참조 무결성 : 릴레이션은 참조할 수 없는 외래키 값을 가질 수 없음

정규화

정규화 : 개념적 설계와 논리적 설계 단계사이에서 수행. 데이터 속성의 중복을 최소화하고 일치성을 보장함

함수 종속성 : 릴레이션의 어떤 속성의 값이 다른 속성의 값을 고유하게 결졍하는 것

이상 : 정규화를 하지 않으면 데이터들이 불필요하게 중복되어 릴레이션 조작시 예기치 못한 곤란한 현상

SQL : 관계대수와 관계해석을 기초로 한 혼합 데이터 언어. 1974년 IBM에서 개발안 SEQUEL에서 유래
관계 해석 : 코드가 개발하여 술어해석에 기반을 둠.
DML : 데이터베이스 사용자와 데이터베이스 관리 시스템간의 인터페이스 제공
DOMAIN 만들때 default 값 을 하면 자동으로 입력댐
RESTRICT : 삭제할 요소를 다른 개체가 참조중일때는 삭제 취소

SELECT 문은 튜플들 중에서 전체 또는 조건을 만족하는 튜플을 검색하여 주기억장치 상에 임시 테이블로 구성하는 명령문

DISTINCTROW : 중복된 튜플을 제거하고 한 개만 검색하지만 선택된 속성의 값이 아닌, 튜플 전체를 대상으로 함.

HAVING절 : 그룹에 대한 조건을 지정

NOT IN : 포함 안할거다

UNION : 두 테이블에 모두 속해 있는 자료는 한 개만 표시한다.

내장SQL : 프로그램이 실행될때 함께 실행되도록 삽입된 SQL
스토어드 프로시저 : 미리 저장해둔 SQL문. 복잡한 SQL의 일괄 작업을 수행하는데 적합하다.
