2019-04-12 Tree

2019/04/12 카테고리 자료구조

Tree

트리는 그래프의 일종이다(Connected acyclic graph)

  • 그래프에서의 순회를 트리에서 적용할 수 있다

정의

  • 연결이 있고 사이클이 있는 그래프
  • 루트 노드(root)가 있다
  • 트리를 구성하는 노드 간에 단순 경로가 존재
    • 이동할때 갔던 경로를 다시 방문하지 않고 갈 수 있으면 단순경로가 있다고 한다
  • Start에서 출발해서 Start로 올수있다 (cyclic)
  • Connected : 경로가 무조건 하나는 있다

Tree 용어

  • 노드의 차수 : 자식의 갯수
  • 트리의 차수 : 노드의 차수 중에 가장 큰것
  • 리프 노드 : 자식이 없는 노드
  • 엣지 : 노드와 노드를 연결하는것
  • 트리의 높이 = 깊이 : 트리의 레벨
  • 트리가 되기위한 조건식 : 엣지 = 노드 - 1
  • 포레스트 : 서브 트리의 집합(Connected 만족 X) -> 트리가 모여있으므로 포레스트
  • 일반적으로 트리를 이야기 하면 이진트리(binary Tree)

Binary Tree

정의

  1. 어떤 노드의 최대 자식 수가 2개이하 인것
  2. 공집합 혹은 루트와 왼쪽 서브트리, 오른쪽 서브 트리로 이루어진 유한집합, 각각의 서브트리는 모두 이진 트리이다
    • 자식이 없지만 있다고 치는 노드를 공노드(Empty Node)라 한다.

특징

  • 각 Level에서의 최대 노드수 : 2^(level-1) 개
  • 트리 전체의 최대 노드 수 : 2^(Tree level) -1 개

포화 이진트리

  • 모든 레벨이 꽉 차 있다.

완전 이진트리

  • 일정 레벨 까지는 꽉 차있고 바로 밑 레벨부터 왼쪽에서 오른쪽으로 채워져있는 트리

편향 이진트리

  • 한쪽으로만 치우친 트리

트리의 순회

  • 재방문을 허용하지 않고 트리를 구성하는 모든 노드를 한번씩 방문하는것
순회의 계열은 2가지로 나뉜다

  • Stack계열

    • DFS(Depth First Search) - 그래프용어 : 깊이 우선 순회
      1. 전위순회(Preorder traversal) : A - B - C의 순서
      2. 중위순회(Inorder traversal) : B - A - C의 순서
      3. 후위순회(Post traversal) : B - C - A의 순서
    • 재귀 : 재귀를 하면 Stack Frame이 쌓임
    • 반복문 + Stack

  • Queue계열

    • BFS(Broath First Search) - 그래프용어: 너비 우선 순회
      • 트리에서는 레벨 순서 순회(level order)

BST(Binary Search Tree)

dictionary 구현하기

  1. hashing -> 파이썬의 dictionary는 hashing으로 구현됨
    • 테이블
  2. BST -> O(log N)
    • 이진트리의 일종
    • 모든 원소는 서로 다른 key를 가진다
    • 왼쪽 서브 트리에 있는 모든 키들은 루트의 키보다 작다.
    • 오른쪽 서브 트리에 있는 모든 키들은 루트의 키보다 크다.
    • 왼쪽 서브 트리와 오른쪽 서브 트리도 이진 탐색 트리이다.
    • 재귀적으로도 만족 해야한다 : 자식 트리역시 똑같은 로직으로 작동해야 한다

사족

  • delete() -> None : 리스트에서도 지워지고 메모리에서 지워짐
  • remove() -> removed data : 리스트에서 지워진 데이터를 반환(데이터를 반환 함으로써 사용자에게 어떻게 처리할건지 결정하게함)
  • python의 dictionary는 immutable 객체만 key로 사용할 수 있다
  • BST는 속도가 빠르기때문에 DB에서 사용하는 B-Tree역시 BST의 일종이다
  • BST는 Binary Tree의일종이므로 만들때는 상속으로 만드는게 좋다

2019-04-11 List와 Array의 차이점

2019/04/11 카테고리 자료구조

리스트(List)와 배열(Array)의 차이점

  1. 동적배열(Dynamic Array) vs 연결리스트(Linked List)의 차이

    • DynamicArray

      • 장점:
        • search : O(1) -> Indexing을 사용하기 때문
          • 메모리에 가져올때 지역성이 적용되어 캐시히트가 일어나기 때문
      • 단점:
        • Insert : O(n) -> 삽입을 하기위해 기존의 n개의 데이터를 뒤로 밀어야 한다
        • Delete : O(n) -> 삭제를 한 뒤 남아있는 데이터를 앞으로 당겨야 한다.

    • LinkedList

      • 장점: 데이터의 삽입, 삭제가 매우 빠르다
        • Insert : O(1)
        • Delete : O(1)
      • 단점:
        • Search : O(n)
          • 지역성이 적용될 확률이 낮기때문에 캐시히트가 일어나기 힘들다

Search를 많이하면 배열을, Insert, Delete를 할때는 연결리스트를 주로 사용한다

다이나믹 힙(Dynamic Heap)기법으로 메모리 할당을 한쪽으로 몰아도 되도록이면 배열을 사용한다

Git

브랜치란?

  • 분기점을 생성하고 독립적으로 코드를 변경할 수 있도록 도와주는 모델

  • Ex)

    • matser branch

    • another branch

  • 로컬의 branch와 리모트의 branch는 따로 존재한다

  • git branch -r : 리모트의 브랜치를 조회한다

  • git branch : 로컬의 브랜치를 조회한다

브랜치 생성

  • git branch feat/loop -> feat/loop라는 새로운 브랜치를 만든다
    • 현재 브랜치의 현재 상태 그대로 복제하여 만든다

브랜치 이동

  • git checkout feat/loop -> feat/loop 브랜치로 이동한다.

리모트 레포와 동기화

  • git push -u origin feat/loop : 리모트 레포에 feat/loop라는 새로운 브랜치 생성
    • -u : upstream의 약자 -> 리모트 레포에 feat/loop가 없으므로 해주어야 한다.

브랜치를 해도 deletion이 없으면 평행으로 되고 있으면 밑으로 추가되고, 우리가 논리적으로 생각하는 것으로 그림을 그린다.

브랜치 병합하기

  1. 우선 병합하고자 하는 브랜치로 이동한다 -> git checkout feat/loop
  2. git merge feat/con-state : feat/loop를 feat/con-state의 내용과 동기화시킨다(feat/con-state와 같아진다)
  3. 자동으로 commit이 생긴다.

브랜치 삭제하기

  • git branch -D/d 브랜치명
    • D는 강제로 삭제한다

브랜치및 머지를 보다 쉽게 하기

  • git flow 전략 : 다운로드 링크

    • master는 항상 개발이 끝난 것만 보여야한다

  • git init -> git flow init : git init을 하면서 flow까지 한꺼번에 함

  • develop 브랜치를 만들어도 기능 개발을 할때는 브랜치를 또 만들어서 해야한다

  • git flow feature start Name : Name의 브랜치를 생성하고 checkout 한다

    • 자식 브랜치를 만들고나서 파일을 수정하고 꼭 commit을 해야 가지가 쳐진다.

  • git flow feature finish Name : Name의 브랜치를 상위 브랜치와 merge하고 Name의 브랜치는 삭제 한다(remote와 local에서 같이 지운다)

    • pattern = r’([0-9]{3})[-./ ](\d{3,4})[-./ ](\d{4})’

2019-04-09 파이썬 Closure

2019/04/09 카테고리 python

파이썬의 함수는 first-class로 취급한다.
함수를 변수에 할당하거나 자료구조 안에 저장할 수 있고 다른 함수의 인자로 사용할 수 있다.

Closure란?

프로그래밍 언어에서의 클로저란 퍼스트클래스 함수를 지원하는 언어의 네임 바인딩 기술이다. 클로저는 어떤 함수를 함수 자신이 가지고 있는 환경과 함께 저장한 레코드이다. 또한 함수가 가진 프리변수(free variable)를 클로저가 만들어지는 당시의 값과 레퍼런스에 맵핑하여 주는 역할을 한다. 클로저는 일반 함수와는 다르게, 자신의 영역 밖에서 호출된 함수의 변수값과 레퍼런스를 복사하고 저장한 뒤, 이 캡처한 값들에 액세스할 수 있게 도와준다.
출처 School Of Web

기존의 함수들은 input이 같으면 output역시 같았다. 그러나 초기값에 따라 input이 같아도 out은 다르다.

  1. my_account라는 변수에 account함수를 호출하여 "김기동"과 5000이라는 값을 넘겨주었다
  2. account 함수에서는 넘겨받은 값들을 사용하지 않고 바로
    return change_money를 해주었다
  3. my_account는 return받은 change_money가 되었다
  1. 3번에서 my_account가 change_money가 되었으므로
    my_account(3000)을 하게되면 change_money(3000)을 하게된것과 똑같다.
    그리하여 결과값은 “김기동”, 8000이 된다.

그렇다면 account함수 내의 change_money함수는 account가 return되어 종료되었음에도 불구하고 account의 지역변수인 clnt_name과 balance에 접근할수 있는걸까? 이것은 클로저는 스택이 아닌 힙에 저장되기 때문이다 힙에 저장된 값들은 사용자가 직접 제거하지 않는 이상 사라지지 않는다.

2019-04-08 데이터베이스 관계, 조인, Group, Index

2019/04/08 카테고리 데이터베이스

N:M 관계

  • A테이블과 B테이블이 있을때 A테이블의 레코드 1개가 B테이블의 레코드 N개로 대응되고 B테이블의 레코드 1개가 A테이블의 레코드 M개와 대응될때 M:N이라 한다.

1:N or N:1관계

  • A테이블 레코드 하나에 B테이블 레코드가 N개 대응될때 1:N이라 한다.

조인

  • inner join (MySQL, Maria DB, Oracle은 Where로 묶어도 잘 작동하지만 ANSI 표준으로 하는게 더 좋다)
  • outer join
    • left outer
      • A from B를 했을때 A, B의 동일한 필드로 묶었을때 A 테이블을 기준으로 표시하고 B필드값이 NULL이더라도 표시한다
    • right outer
      • A from B 를 했을때 A와 B가 동일한 필드로 묶었을때 B 테이블을 기준으로 A의 필드값이 NULL이더라도 표시한다
    • full outer(MySQL, Maria DB 지원 X -> UNION이라는 명령어를 통해 구현)
  • 테이블의 데이터를 가로로 연결한다 (레코드대 레코드)
  • 일반적으로는 join하면 inner join 이다

Group By

  • Where와 Having의 차이
    • Where : 그룹핑 이전에 조건을 수행함
    • Having : 이미 Group By로 그룹을 나눈후에 조건을 수행함 -> Group By가 반드시 필요함

schema :DB의 뼈대

VIEW

  • 필요할때마다 select문을 계속 작성을 할수 없으므로 view를 하나 설정해 마치 테이블처럼 사용하는 것
  • view는 테이블이 새로 만들어 지는것이 아님
  • 가독성이 높아진다.
  • 보안성

INDEX

필드를 기준으로 데이터 검색을 할때 사용한다.
인덱싱을 제외하고 트리구조( 그 중에서도 BST(log N)가 가장 빠르다
인덱스는B Tree로 함

index를 지우려면 foreign key를 먼저 지우고 index를 지운다

  • ALTER TABLE 테이블명 DROP FOREIGN KEY 외래키명 ;

index는 필요없어지면 바로 지워주는게 좋다 (성능 하락에 주범)

  • DROP INDEX 인덱스명
  • CLUSTERED INDEX
    • 한 테이블에 1개씩만 존재함
      • Primary Key
      • Unique Not NULL
    • 데이터 자체를 클러스터 인덱스에 맞춰 정렬을 해 저장을 한다 -> 이것으로 인해 한테이블에 1개씩밖에 존재함
      • 주키를 잡으면 자동으로 클러스터 인덱스로됨(반드시 테이블을 만들때 주키를 잡을것)
      • ALTER TABLE 명령어를 사용해야
    • DROP INDEX PRIMARY ON students; -> 주키를 제거하면서 인덱스 역시 제거
    • ALTER TABLE students ADD CONSTRAINT PRIMARY KEY(studentID) -> students 테이블에 studentID속성을 주키로 설정하겠다
  • SECONDARY INDEX
    • 장점
      • 검색속도가 엄청나게 빨라짐
      • 쿼리에 대한 부담이 줄어든다
    • 단점
      • insert와 update를 계속된다면 테이블이 계속 갱신이 되어야 하므로 페이지 분할이 일어나 속도가 매우 느려진다.
      • 메모리를 잡아먹음 약 10%
    • 원하는 만큼 얼마든지 가능
    • B 트리 라는 자료구조를 만들고 이것이 데이터를 참조하는 형식 (Where 을 할때 인덱스가 있다면 쓰인다) -> Page와 Offset으로 구성되어있음
    • Where를 자주쓰는 상황이면 만드는게 좋다. 만약 사용을 하지않게 된다면 무조건 지워라

2019-04-07 TCP/IP 4계층

2019/04/07 카테고리 네트워크

OSI 7 Layer 와 TCP/IP

  • 응용 계층(Application Layer) = Application
  • 표현 계층(Presentation Layer)
  • 세션 계층(Session Layer)
  • 전송 계층(Transport Layer) = Transport
  • 네트워크 계층(Network Layer) = Internet
  • 데이터 링크 계층(Data link Layer) = Network Interface
  • 물리 계층(Physical Layer)

TCP/IP에서 알아야할 프로토콜

응용 계층(Application Layer) = FTP, HTTP, SSH
전송 계층(Transport Layer) = TCP, UDP
네트워크 계층(Network Layer) = IP
데이터 링크 계층(Data link Layer) = Ethernet

TCP/IP 4계층

네트워크 전송 시 데이터 표준을 정리한것이 OSI 7 Layer이고 이 것을 바탕으로 실제 사용하는 인터넷 표준이 TCP/IP 4계층이다.

4 계층 : Application (OSI 7Layer의 5, 6, 7계층 담당)

  • 주요 프로토콜
    • FTP
    • Telnet : 보안에 너무 취약하여 현재는 안쓰임
    • DNS(Domain name System) : IP와 Name(Domain)을 이어주는 역할
      • 사용자가 보내면 request를 받아야하는데 이 역할을 하는게 DNS server
    • HTTP
      • http://www.naver.com///~/
      • schema/host/ /디렉토리들/
      • 1.1 부터는 clnt랑 server랑 한번 연결하면 필요없을때까지 연결을 끊지 않음

3계층 : Transport (OSI 7Layer의 4계층 담당)

port(socket에 부여된 번호, port는 프로세스안에 있음)번호를 관리함

  1. TCP(E-mail, 채팅)
    • 신뢰도 높음
      • timeout이 났을 경우 자동으로 재전송
    • 연결 지향적
      • 상대방에게 패킷을 보내고 받았는지 체크(ACK)
    • TCB
      • 서로 연결이 되면 TCB에 등록을 해놓음
      • 소켓쌍 - (source IP, source port), (dest IP, dest port)
      • send buffer, receive buffer에 대한 포인터
        • send와 receive -> 커널 영역
    • sliding window
      • receive window : 내가 받을 수 있는 패킷의 양
    • 3 Way handShaking
      • byte stream에 첫번째 부여하는 숫자를 ISN(initial sequence number)이라 한다
      • 그 숫자를 sequence라고 함
        1. client가 server에 SYC와 자기가 보내고자한 바이트의 seq를 같이 보냄
        1. server역시 SYC + ACK(상대방이 보낸 seq에 +1을 해서)를 보내 자신의 send buffer와 client의 receive buffer와 동기화 시킴
    • TearDown
        1. client가 FIN를 보냄
        1. Server는 ACK를 보냄 -> clinet는 send는 못하지만 receive는 가능, server가 남은 데이터를 send함
        1. 남은 데이터를 보내고 난뒤 server가 FIN를 보냄 (Time Out만큼 기다림)
        1. client가 server의 FIN을 받았으므로 ACK을
  2. UDP(스트리밍 서비스, 게임)
    • 신뢰도 낮음
      • 상대방에게 패킷을 보내도 받았는지 체크 안함
    • 비 연결 지향형 프로토콜
    • 데이터를 재전송하지 않는다.
    • 전송된 데이터 일부가 손실될 수 있다.

2계층 : Internet(OSI 7Layer의 3계층 담당) -> IP routing을 함, IP address를 다루는 프로토콜

  • IP routing : source에서 desti로 갈때 어느 router로 갈지 결정하는 것
    • 라우팅 테이블을 통하여 어느 route로 갈지 등록을 해놓는다
    • netstat -r을 했을때 나오는 Destination은 네트워크 IP이다
    • default는 라우터의 IP
    • 한번 ARP로 하면 같은 LAN 상에서는 서로의 MAC주소를 알고 있으니 게이트웨이가 MAC으로 잡힌다
  • ISP(Internet Service Provider) : KT, SK같은 인터넷 공급자들
  • IP
    • 하나의 IP주소에는 Network ID와 Host ID가 존재하고있음
    • Source Address : 4 bytes, 송신 IP
    • Destination Address : 4 bytes, 수신 IP
    • TTL(Time To Leave) : router를 하나씩 거칠때마다 줄어들어 0이 되면 패킷은 버려진다
    • IP는 network 주소와 host주소로 구분된다
      • network주소 : router를 찾는 주소(LAN)
      • host주소 : 찾은 LAN에 소속되어있는 특정 host를 찾는 주소(=컴퓨터)
    • Subneting
    • public IP(공인 IP) -> IP를 돈을 주고 삼(SK, KT 등)
    • private IP
      • Ex) 커피숍에 공인IP하나를 할당해주고 router가 허용하지 않은 대역대 내에서 마음대로 나머지 고객들에게 private IP를 할당함
      • public IP와 private IP를 통신하게 해주는 것이 NAT, NAPT
        • NAT : 사설 IP로 연결되어있는 IP가 외부로 갈경우 매핑되어있는 테이블과 IP를 바꿔 보내고 다시 받으면 테이블의 IP로 다시바꾸고 그것을 다시 ARP로 바꿔 송신받는 기술
        • NAPT(Network Address Port Translation) : NAT에 추가하여 port번호까지 전송
  • BroadCast : LAN상에 있는 모든 사람들에게 패킷을 보내 MAC주소를 알아냄
    • ARP(Address Resolution Protocol)(프로토콜) -> ip를 MAC주소로 변환
      • 같은 LAN 상에있는 호스트들에게 ping요청을 보내 호스트들의 ip와 MAC주소를 매핑시키는 작업
      • Network Interface와 Internet 사이에 있음

1계층 : Network Interface(OSI 7Layer의 1,2계층 담당) : MAC주소

  • Tx에서 쏜 데이터를 Rx에서 받아야함(Serial 통신) -> Data Link 계층
  • NIC(network interface card) : 일반적으로 랜카드라 불림. 네트워크 어댑터
  • MAC(Media Access Control) : NIC의 하드웨어 주소, 앞의 3자리는 제조사 뒤 3자리는 NIC 번호
  • Ethernet(프로토콜) -> lan에서 host를 구별하기위함
    • Destination MAC Address : 6bytes, 패킷을 수신 하는 NIC -> LAN에서의 router
    • Source Mac Address : 6bytes, 패킷을 송신하는 NIC
    • MTU(한번에 보낼수 있는 데이터의 양) -> 1500bytes : 헤더를 제외하면 1350정도

LAN(Local Area Network) : 보통 router를 기준으로 묶여있는 것(같은 router안에서 host를 구분하기위해 MAC주소를 사용함), 같은 LAN상에서 데이터를 보낼때도 이용
WAN(wide Area Network) : IP를 사용

하드웨어 주소 = 물리주소 = MAC주소 = NIC주소 = LAN카드 주소 = 제2계층주소 = 데이터링크계층 주소
이더넷 주소는 하드웨어주소의 일종입니다.

논리주소 = 네트워크 주소 = 제3계층주소 = 프로토콜 주소
IP주소는 논리주소의 한 종류입니다.

2019/04/07 카테고리 운영체제

Synchronous(동기)와 Asynchronous(비동기)의 차이점

동기식과 비동기식의 큰 차이점은 데이터를 받는 방식에 있습니다

동기식 처리는 A,B,C 순서대로 작업이 수행되었을 경우 A,B,C의 순서로 작업이 끝나야 합니다. 작업의 순서가 보장됩니다.

Synchronous란 '동시에 일어난다’라는 뜻을 가지고 있으며 여기서 동시에 일어난다는 것은 요청에 따른 응답이 동시에 일어난다는 것입니다.


출처 : https://nesoy.github.io/articles/2017-01/Synchronized

반면 비동기식 처리는 작업의 순서가 보장되지 않습니다. A,B,C의 순서로 작업이 시작되었더라도 A,B,C의 순서로 작업이 끝난다고 보장 할 수 없습니다.


출처 : https://nesoy.github.io/articles/2017-01/Synchronized
