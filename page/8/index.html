2019-04-22 Python 가상환경 구축하기

2019/04/22 카테고리 python

우분투에서 python 가상환경 구축하기

제 우분투 버전은 18.04LTS이며 ZShell을 사용하였습니다.
IDE는 Pycham을 기준으로 작성하였습니다.

  1. 파이썬 프로젝트를 생성합니다.

  2. 하단의 터미널을 켜서 pip3 install virtualenv 를 치고 설치해 줍니다

    • 저는 파이썬 기본설정이 Python2이기 때문에 pip3라고 하였습니다. 기본설정이 Python3이면 pip라고 하셔도 무방합니다.

  3. 설치 후 잘 설치가 되었는지 확인하기 위해 virtualenv --version을 입력합니다.

  4. 위와 같이 버전이 뜨면 정상적으로 설치가 되었습니다. 그 다음 가상환경을 만들어 보겠습니다. virtualevn [가상환경명] 을 적어줍니다


    적고 실행시키면 목록이 뜨게 됩니다.

  5. 그럼 이제 가상환경을 실행해 보겠습니다.
    source [가상환경명]/bin/activate를 실행합니다.

    실행 하고나면 이런식으로 맨 왼쪽에 [가상환경명]이 뜨게 되며 적용이 된것을 확인할 수 있습니다.
    이 상태에서 **pip3 install [패키지명]**을 통해 패키지를 설치하시면 됩니다.

    • [패키지명]==[버전]을 하게되면 원하는 버전으로 설치가 가능합니다.

  6. 가상환경을 종료하시려면 deactivate를 실행해 줍니다.

2019-04-21 localhost와 127.0.0.1의 차이점

2019/04/21 카테고리 네트워크

127.0.0.1은 무엇인가?

127.0.0.1은 IPv4 및 IPv6에서 자기 자신을 가리키기 위한 목적으로 쓰기 위해 예약된 IP이다. 모든 컴퓨터는 이 주소를 자신의 주소로 사용하지만 실제 IP 주소처럼 다른 장치와 통신 할 수는 없다.

  • Ex) 컴퓨터에서 실행중인 웹 서버는 127.0.0.1을 가리킬 수 있으므로 페이지를 배포하기 전에 로컬에서 실행하고 테스트 할 수 있다.

LocalHost는 무엇인가?

localhost은 자기 자신을 가리키는 IP의 도메인 네임으로써 알기 쉽게 해놓은 것이다. 그러므로 localhost로 접속을 하게되면 DNS의 테이블과 비교를 해야하는 번거로움이 있다 그로 인하여 127.0.0.1로 직접 접속하는것 보다 느리지만 기억하기는 쉽다.
그리고 localhost는 hosts 파일을 통해 변경될 수 있다

루프백

호스트 자신을 가리키는 주소이다 127.0.0.1을 사용한다. 루프백 주소로 네트워크에 직접 연결하지 않은 상태에서 빠르게 프로그램의 테스트를 할 수 있다. 또한 루프백 IP를 갖는 패킷은 외부 네트워크로 전송할수 없다

Django 기본

Django 프로젝트 구조 살펴보기

  1. second_django 폴더 : 프로젝트 설정 파일과 웹 서비스 실행을 위한 파일들이 들어있다.

    • init.py : 파이썬 2.x대 버전과의 호환을 위해 만들어진 비어있는 파일이다. 지워도 무방하다
    • settings.py : 프로젝트 설정에 관한 다양한 내용이 들어있다.
    • urls.py : 특정 기능을 수행하기 위해 접속하는 주소를 url이라 하고 이를 설정해주는데, 그 설정을 이 파일에 기록함
    • wsgi.py : 웹 서비스를 실행하기 위한 wsgi 관련 내용이 들어있음

  2. venv 폴더 : 프로젝트 구동에 필요한 가상환경이 들어있음

  3. db.sqlite3 : SQLite3 DB 파일이다. 다른 DB를 이용할시 필요없다

장고 기본 명령들

  1. django-admin startproject : 장고 프로젝트를 만드는 명령이다. 웹 서비스를 만들 때 마다 한번 실행한다.
  2. startapp : 프로젝트에 기능 단위인 앱을 새로 만들때 사용
  3. makemigrations : 어플리케이션에 변경 사항을 추적해 DB에 적용할 내용을 정리함. 보통 앱 안에 있는 모델(model)의 변경 사항이 있을 때 주로 사용함.
  4. sqlmigrate: 실행할 SQL 명령문을 출력한다
  5. migrate : 실제 변경사항을 DB에 반영한다.
  6. showmigrations : 프로젝트의 DB 변경사항 목록과 상태를 출력한다.
  7. runserver : 테스트 서버를 실행한다.
  8. dumpdata : 현재 DB의 내용을 백업할 때 사용한다.
  9. loaddata : 백업 파일에서 DB로 내용을 복구 할 때 사용한다
  10. flush : DB 테이블은 그대로 두고 테이블의 내용만 전부 삭제한다
  11. shell : 장고 쉘(Shell)을 실행한다
  12. dbshell : DB에 직접 접근 할 수 있는 쉘을 실행한다
  13. createsuperuser : 관리자 계정을 생성한다
  14. changepassword : 계정의 비밀번호를 변경할 수 있다

2019-04-18 프로세스와 쓰레드

2019/04/18 카테고리 운영체제

Program, Process

Program => HDD안의 이미지를 말함 Ex) Python.exe

Process => 실행중인 프로그램(이미 RAM에 올라가있음)

PID로 구분을 한다 (Process ID) -> 윈도우는 랜덤으로, 리눅스는 순차적으로 배분

pre-emptive OS

  • multitasking OS
  • suspend currently running process
  • determine which process should execute next

process scheduling

  • priority algorithm : 높은 우선순위의 프로세스가 먼저 실행된다.
  • round-robin algorithm : 프로세스들 사이에 우선순위를 두지 않고 time slice로 CPU를 할당하는 방법이다.

scheduler

  • When?
    • every time slice
    • a process created or a process ended
    • a running process blocked (I/O, system call)

Block으로 빠지는 경우

  • IO
  • System call(운영체제에서 사용하는 함수중 사용자에게 인터페이스를 열어놓은 함수 Ex) sleep)

Waiting 과 Running 사이에서 일어나는 일 : context switching

context switching

  • 메모리안에 블록을 하나만듬(Process Control block) 이곳(PCB)에 어디까지 실행하였는지(CPU안의 레지스터들의 상태정보)를 저장함
  • context = 지금 현재 실행되고있는 CPU안의 레지스터들의 상태정보를 하나로 모은것
  • 자주일어날경우 메모리접근 시간이 너무 커짐 -> time slice를 잘 정해야함
  • PCB를 내리고 새로운 PCB를 올리는 작업을 context switching이라 함

Process와 Thread -> 인스트럭션의 나열 -> 실행의 흐름일 뿐이다

Process = 1개 이상의 쓰레드를 갖는다, 쓰레드가 1개일경우 (single thread라 한다)

Concurrency programming (동시성 프로그래밍) -> 속도를 위해 하는것이 아님(싱글 코어일 경우)

  1. multi - processing

    • 각기 다른 프로세스에서 똑같은 데이터를 가지고 있어야함
    • IPC(Inter Process Communication) -> 최악의 효율성

  2. multi - threading

    • 프로세스안에 스레드가 포함이 됨(스택만 나눠지고 나머지는 공유함)
    • 스택 영역만 스레드마다 따로잡힘(지금 현재 실행하는 인스트럭션(함수)이 다르므로 스택만 나누고 나머지 공간은 공유가능)
    • 쓰레드 또한 PCB처럼 TCB가 있고 context switching이 똑같이 일어남

  3. I/O-bound -> asynchronus I/O(비동기 I/O)

    • 한개의 스레드로 동시성으로 만듬

GIL -> Global Interpreter Lock

  • 멀티코어라도 바이트 코드의 실행을 한번에 하나 스레드로 막는다 (없애려고 했으나 해결을 하지 못함) -> Cpython은 아직 해결 못함

race-condition

  • 공유자원에 대해 너도나도 쓰겠다고 달려들어 결과값을 보장하지 못한다.

임계 구역(critical section) -> 접근하는것 까지는 괜찮으나 수정하는건 안됨

mutual exclution(상호배제)

  • 세마포어 : 한번에 임계구역에 접근할 수 있는 스레드의 숫자를 조정함
  • mutex : 임계구역에 단 하나밖에 접근 못함

코어가 1개 일 경우 시간 단축은 거의 없지만(concurrecy programming)
코어가 4개 일경우 시간단축 + 멀티코어 프로그래밍이 같이됨
concurecncy programming과 pararrelism programming은 다른 개념이다

2019-04-16 실수의 표현

2019/04/16 카테고리 컴퓨터구조

실수의 표현 (epsilon comparison을 위해 반드시 알아두어야함)

  1. 고정 소수점 -> 표현숫자를 줄이면 정밀도가 높아아지고 늘리면 정밀도가 낮아짐, 표현범위가 좁음 #현재 잘 사용하지 않음

  2. 부동 소수점 -> 정밀도 매우낮음, 표현범위 넓음

    1. 단정도 (single precision)
      1. 32bit (4 byte)
      2. 부호(sign) (1 bit) + 지수(exponent) (8 bit) + 가수(mentissa) (23 bit)
        1. 지수부 : 표현범위 담당 (클수록 범위)
        2. 가수부 : 정밀도를 담당 (클수록 정확)
    2. 배정도 (double precision)
      1. 64bit (8 byte)
      2. 부호 (1 bit) + 지수 (11 bit) + 가수 (52 bit)

  3. 10.23 * 102 -> 1.023(가수부) 10(밑수) 3(지수 = exponent real)

  4. 에서 1도 수학적으로는 mentissa이나 컴퓨터 메모리 저장 X --> 0이 아닌 자연수 1자리로 정규화를 하기위해 한자리는 무조건 남겨둠 /// bias = 2n-1 -1 /// Ere = Emem - bias /// float의 bias는 127 --> 지수에도 ±기호가 있으므로 맨앞자리 비트는 제외하고 나머지 1일때 127 // bias + exponent real 한 값을 이진수로 바꿔서 exponent에 저장

    • 정규화 : 가수 부분을 밑수보다 작은 자연수가 되도록 만드는것

  5. 실수의 정밀도 보장 : float는 6자리, double 15자리의 의미 : 10진수 1자리를 다 표현하려면 2진수 4자리(0~15), 10진수 2자리를 다 표현하려면 2진수 7자리(0~127), 10진수 3자리 -> 2진수 10자리

    1. float는 가수부가 23+1자리이므로 이진수 23+1자리라고 생각가능 -> 십진수 6자리표현을 보장함(정밀도)
    2. double은 가수부가 52+1 자리 이므로 이진수 52+1자리라고 생각 가능 -> 십진수 15자리 표현을 보장함

  6. 1.0과 1.0다음으로 나타낼수 있는 수의 차이를 epsilon이라 함

    • 어떠한 수의 지수부분과 엡실론을 곱하면 그 수의 다음수가 나옴

2019-04-15 알고리즘이란?

2019/04/15 카테고리 알고리즘

알고리즘이란?

  • 유한한 단계를 통해 문제를 해결하기 위한 절차나 방법

알고리즘 표현방법

  • 슈도코드 : 특정 프로그래밍 언어의 문법을 따라 쓰여진 것이 아니라, 일반적인 언어로 코드를 흉내내어 알고리즘을 써 놓은 코드
    • 의사 코드로 흉내낸 코드
    • 실제적인 프로그래밍 언어로 작성된 코드처럼 컴퓨터에서 실행할 수 없음
    • 특정 언어로 프로그램을 작성하기 전에 알고리즘을 대략적으로 모델링하는데 쓰임
  • 순서도 : 프로그램이나 작업의 진행 흐름을 순서에 따라 여러 가지 기호나 문자로 나타낸 도표

무엇이 좋은 알고리즘인가?

  • 정확성 : 얼마나 정확하게 동작하는가?
  • 작업량 : 얼마나 적은 연산으로 원하는 결과를 얻어내는가?
  • 메모리 사용량 : 얼마나 적은 메모리를 사용하는가?
  • 단순성 : 얼마나 단순한가?
  • 최적성 : 더 이상 개선할 여지 없이 최적화 되어있는가?

알고리즘의 성능 분석

  • 많은 문제에서 알고리즘의 성능 분석기준으로 알고리즘의 작업량을 비교
    • Ex) 1~100까지의 합을 구하는 문제
    1. 실제 걸리는 시간을측정
    1. 실행되는 명령문의 개수를 계산 -> 일반적으로 많이 사용됨
    1. Big-O 표기법 : 시간 복잡도 함수 중에서 가장 큰 영향력을 주는 n에 대한 항만을 표시, 계수는 생략하여 표시
    • 요소 수가 증가함에 따라 각기 다른 시간 복잡도의 알고리즘은 아래와 같은 연산 수를 보임

2019-04-13 Greedy Algorithm

2019/04/13 카테고리 알고리즘

greedy algorithm

  • 각 단계마다 지금 당장 가장 좋은 방법만을 선택

  • 시간이나 공간적 제약으로 인해 다른 방법으로 최적해를 찾기 너무 어려운 경우 최적해 대신 근사해로 대체한다

예시

  • 최단경로

정의

  • 연결이 있고 사이클이 있는 그래프
  • 루트 노드(root)가 있다
  • 트리를 구성하는 노드 간에 단순 경로가 존재
    • 이동할때 갔던 경로를 다시 방문하지 않고 갈 수 있으면 단순경로가 있다고 한다
  • Start에서 출발해서 Start로 올수있다 (cyclic)
  • Connected : 경로가 무조건 하나는 있다

Tree 용어

  • 노드의 차수 : 자식의 갯수
  • 트리의 차수 : 노드의 차수 중에 가장 큰것
  • 리프 노드 : 자식이 없는 노드
  • 엣지 : 노드와 노드를 연결하는것
  • 트리의 높이 = 깊이 : 트리의 레벨
  • 트리가 되기위한 조건식 : 엣지 = 노드 - 1
  • 포레스트 : 서브 트리의 집합(Connected 만족 X) -> 트리가 모여있으므로 포레스트
  • 일반적으로 트리를 이야기 하면 이진트리(binary Tree)

Binary Tree

정의

  1. 어떤 노드의 최대 자식 수가 2개이하 인것
  2. 공집합 혹은 루트와 왼쪽 서브트리, 오른쪽 서브 트리로 이루어진 유한집합, 각각의 서브트리는 모두 이진 트리이다
    • 자식이 없지만 있다고 치는 노드를 공노드(Empty Node)라 한다.

특징

  • 각 Level에서의 최대 노드수 : 2^(level-1) 개
  • 트리 전체의 최대 노드 수 : 2^(Tree level) -1 개

포화 이진트리

  • 모든 레벨이 꽉 차 있다.

완전 이진트리

  • 일정 레벨 까지는 꽉 차있고 바로 밑 레벨부터 왼쪽에서 오른쪽으로 채워져있는 트리

편향 이진트리

  • 한쪽으로만 치우친 트리

트리의 순회

  • 재방문을 허용하지 않고 트리를 구성하는 모든 노드를 한번씩 방문하는것
순회의 계열은 2가지로 나뉜다

  • Stack계열

    • DFS(Depth First Search) - 그래프용어 : 깊이 우선 순회
      1. 전위순회(Preorder traversal) : A - B - C의 순서
      2. 중위순회(Inorder traversal) : B - A - C의 순서
      3. 후위순회(Post traversal) : B - C - A의 순서
    • 재귀 : 재귀를 하면 Stack Frame이 쌓임
    • 반복문 + Stack

  • Queue계열

    • BFS(Broath First Search) - 그래프용어: 너비 우선 순회
      • 트리에서는 레벨 순서 순회(level order)

BST(Binary Search Tree)

dictionary 구현하기

  1. hashing -> 파이썬의 dictionary는 hashing으로 구현됨
    • 테이블
  2. BST -> O(log N)
    • 이진트리의 일종
    • 모든 원소는 서로 다른 key를 가진다
    • 왼쪽 서브 트리에 있는 모든 키들은 루트의 키보다 작다.
    • 오른쪽 서브 트리에 있는 모든 키들은 루트의 키보다 크다.
    • 왼쪽 서브 트리와 오른쪽 서브 트리도 이진 탐색 트리이다.
    • 재귀적으로도 만족 해야한다 : 자식 트리역시 똑같은 로직으로 작동해야 한다

사족

  • delete() -> None : 리스트에서도 지워지고 메모리에서 지워짐
  • remove() -> removed data : 리스트에서 지워진 데이터를 반환(데이터를 반환 함으로써 사용자에게 어떻게 처리할건지 결정하게함)
  • python의 dictionary는 immutable 객체만 key로 사용할 수 있다
  • BST는 속도가 빠르기때문에 DB에서 사용하는 B-Tree역시 BST의 일종이다
  • BST는 Binary Tree의일종이므로 만들때는 상속으로 만드는게 좋다

리스트(List)와 배열(Array)의 차이점

  1. 동적배열(Dynamic Array) vs 연결리스트(Linked List)의 차이

    • DynamicArray

      • 장점:
        • search : O(1) -> Indexing을 사용하기 때문
          • 메모리에 가져올때 지역성이 적용되어 캐시히트가 일어나기 때문
      • 단점:
        • Insert : O(n) -> 삽입을 하기위해 기존의 n개의 데이터를 뒤로 밀어야 한다
        • Delete : O(n) -> 삭제를 한 뒤 남아있는 데이터를 앞으로 당겨야 한다.

    • LinkedList

      • 장점: 데이터의 삽입, 삭제가 매우 빠르다
        • Insert : O(1)
        • Delete : O(1)
      • 단점:
        • Search : O(n)
          • 지역성이 적용될 확률이 낮기때문에 캐시히트가 일어나기 힘들다

Search를 많이하면 배열을, Insert, Delete를 할때는 연결리스트를 주로 사용한다

다이나믹 힙(Dynamic Heap)기법으로 메모리 할당을 한쪽으로 몰아도 되도록이면 배열을 사용한다
