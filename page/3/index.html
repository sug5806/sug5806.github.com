TCP

TCP는 신뢰성 있는 통신이 필요로 한 애플리케이션을 위해 점대점 채널을 제공합니다. 하이퍼텍스트 전송 프로토콜(HTTP), 파일 전송 프로토콜(FTP) 및 텔넷은 신뢰할 수 있는 통신 채널이 필요로 한 애플리케이션입니다. 데이터가 네트워크를 통해 전송 및 수신되는 순서는 이들 애플리케이션의 성공에 매우 중요합니다. HTTP로 URL에서 데이터를 읽을 때 데이터는 보내진 순서대로 수신되어야 합니다. 그렇지 않으면 뒤죽박죽된 HTML 파일, 손상된 압출 파일 또는 다른 잘못된 정보를 받게 됩니다.

  • Application: HTTP, FTP, telnet, SMTP 등
  • Transport: TCP, UCP 등
  • Network: IP 등
  • Link: Device driver 등

전송 프로토콜(Transport protocols)은 하나의 포트에서 다른 포트로 정보를 전달해 애플리케이션 프로그램 간을 통신할 수 있게 하기 위해 사용됩니다. 이들은 연결지향 또는 비연결지향 통신 방식 중 하나를 사용합니다. TCP는 연결 지향형 프로토콜이며 UDP는 비연결 지향형 전송 프로토콜입니다.

연결 지향형 프로토콜 TCP는 출발지 포트/IP 주소와 도착지 포트/IP 주소 간의 통신 연결을 수립합니다. 연결이 종료되고 링크가 끊어질 때까지 포트들은 이 링크를 통해 서로 연결됩니다. 연결 지향 프로토콜의 한 예가 전화 통화입니다. 전화 연결이 수립되어 통신이 이루어진 다음 연결이 종료됩니다.

MVC 디자인 패턴이란?

모델-뷰-컨트롤러(MVC)는 구조적 패턴입니다. MVC 디자인 패턴에서, 응용 프로그램은 모델(Model), 뷰(View), 컨트롤러(Controller)로 알려진 컴포넌트로 나뉘어 집니다. 이 패턴의 목표는 응용 프로그램의 입력(Controller), 비즈니스 처리 로직(Model) 그리고 출력 로직(View)로 나누는 것입니다.

  • 컨트롤러는 모델과 뷰에 사용자 입력과 연관됩니다.
  • 모델은 저장소에서 표현될 데이터를 불러옵니다.
  • 뷰는 불러온 데이터를 어떻게 사용자에게 표현할 것인가를 다룹니다

컨트롤러는 사용자와 처리(Model) 그리고 서식(View)로직 사이의 중간자로 생각할 수 있습니다. 모든 어플리케이션에 대한 모든 사용자 요청 또는 입력의 진입 지점입니다. 컨트롤은 사용자의 입력을 받아들여 그것을 분석하고 어떤 모델과 뷰를 호출할 것인지를 결정합니다. 이에 따라 요청된 정보를 사용자에게 제공하기 위해 선택된 모델과 뷰를 호출합니다.

모델은 응용프로그램의 비즈니스 처리 로직을 나타냅니다. 이것은 응용프로그램 로직을 캡슐화한 것입니다. 모델은 또한 데이터베이스와 연동하고 삽입, 갱신 그리고 삭제 연산을 수행할 책임이 있습니다.

모든 모델은 호출된 후 컨트롤에 특정 종류의 데이터를 제공합니다. 더 나아가 단일 모델이 호출된 모델의 메서드에 기반해 동일한 종류의 데이터에 대한 여러 가지 다른 변형을 반활할 수 있습니다. 정확히 컨트롤에 반환되는 것은 모델의 메서드에 전달되는 인자들로 제어할 수 있습니다.

하노이의 탑이란?

하노이의 탑은 퍼즐의 일종입니다. 세 개의 기둥과 이 기둥에 꽂을 수 있는 크기가 다양한 원판들이 있고, 퍼즐을 시작하기 전에는 한 기둥에 원판들이 작은 것이 위에 있도록 순서대로 쌓여 있습니다.
게임의 목적은 다음 두 가지 조건을 만족하게 하면서, 한 기둥에 꽂힌 원판들을 그 순서 그대로 다른 기둥으로 옮겨서 다시 쌓는 것입니다.

  1. 한 번에 하나의 원판만 옮길 수 있습니다.
  2. 큰 원판이 작은 원판 위에 있어서는 안 됩니다.

2019-08-01 검색

2019/08/01 카테고리 알고리즘

개인적으로 공부한 것을 정리하였습니다.

검색이란?

컴퓨터 공학에서 말하는 검색이란 어떤 요소들의 집합에서 특정 속성을 가지는 요소를 찾는 과정을 말합니다. 요소들은 데이터베이스의 레코드, 배열의 요소, 파일의 텍스트, 트리의 노드, 그래프의 정점 혹은 간선이거나 또는 다른 검색 공간의 기타 요소들일 수 있습니다.

검색의 종류

  • 불규칙 선형 검색
  • 정렬/규칙 선형검색
  • 이진 검색

불규칙 선형 검색

배열의 요소들이 정렬되어 있지 않으면 어떤 요소를 찾고 싶다면 주어신 배열에 해당 요소가 있는지를 모든 요소에 대해 검사해야 합니다.

시간 복잡도 : 최악의 경우 배열의 모든 요소를 검사해야 하기 때문에 O(n)

정렬/규칙 선형 검색

배열이 정렬되었을 경우 해당 배열이 찾고자 하는 요소를 가지고 있는지 여부를 알기 위해 배열의 모든 요소를 조사할 필요가 없습니다.

시간 복잡도 : 최악의 경우 배열의 모든 요소를 검사해야 하기 때문에 O(n)

이진 검색

사전에서 어떤 단어를 찾는다고 생각해보면, 일반적으로 인접한 페이지를 펴서 거기서부터 찾기 시작합니다. 그리고 우리가 찾는 단어를 찾으면 검색은 끝이 납니다. 펼친 페이지가 찾고자 하는 단어 이전이면, 나머지 페이지들을 가지고 처음 과정을 반복하고 그렇지 않으면 앞 페이지에 처음 과정을 반복합니다.

퀵 정렬(Quick) Sort)이란?

퀵 정렬 또한 분할 정복 알고리즘 기술의 좋은 예입니다. 퀵 정렬은 분할 교한 정렬이라고도 하며 요소들을 정렬하기 위해 재귀 호출을 사용합니다.

알고리즘

재귀 호출 알고리즘은 4단계로 구성됩니다.

  1. 배열에 정렬될 요소가 하나 이하일 때 return 합니다.
  2. 피봇(pivot) 지점 역할을 하는 요소를 배열에서 선택합니다
  3. 배열을 두개의 부분으로 분할합니다. 하나는 피봇 값 보다 큰 요소들이고 다른 하나는 피봇 값 보다 작은 요소들이 됩니다.
  4. 원래 배열을 양분한 두 배열에 대해 재귀적으로 알고리즘을 반복합니다.

합병 정렬(Merge Sort)이란?

정렬 과정에서 분학 정복 알고리즘을 사용하기 때문에 하위 문제들을 어떻게 구성해야 하는지 결정해야 합니다. 궁극적인 목표는 전체 배열을 정렬 하는 것입니다.

알고리즘

일반적인 방법으로 구현했을때 안정 정렬에 속하며, 분할 정복 알고리즘의 대표적인 예입니다

  • 분할정복(divide and conquer)방법
    • 문제를 작은 2개의 문제로 분리하고 각각을 해결한 다음, 결과를 모아서 원래의 문제를 해결하는 방법입니다.
    • 분할 정복은 일반적으로 재귀 호출을 이용하여 해결합니다.

삽입 정렬(Insertion Sort)이란?

삽입 정렬은 간단하면서도 효과적인 비교 정렬법입니다. 이 알고리즘은 반복마다 뒤쪽의 정렬되지 않은 요소들의 목록에서 요소를 삭제하고 삭제한 요소를 앞쪽의 이미 정렬된 목록 내 알맞은 자리에 삽입합니다. 입력 데이터에서 삭제될 요소는 정렬되지 않은 목록에서 무작위로 선택하고(일반적으로 앞에서 차례로 선택합니다) 입력 데이터 내 모든 요소를 처리할 때까지 반복합니다.

장점

  • 간단합니다.
  • 적은 데이터 양에 효과적입니다.

선택 정렬(Selection Sort)이란?

선택 정렬은 제자리 정렬 알고리즘입니다. 선택 정렬은 작은 양의 데이터에 유용합니다. 이 알고리즘은 키는 작고 값은 큰 데이터에 사용됩니다. 키를 기반으로 선택이 이루어지고 치환이 필요하다고 판단되면 치환이 이루어지기 때문입니다.

장점

  • 구현이 쉽습니다.
  • 제자리 정렬(추가 저장 공간이 필요없음) 알고리즘입니다.

단점

  • 데이터의 양에 유연하지 않습니다.

알고리즘

  1. 목록에서 최소값을 찾습니다.
  2. 찾은 최소값을 맨 앞의 값(혹은 현재 위치의 값)과 교체합니다.
  3. 배열의 전체 요소가 정렬될 때까지 이 과정을 반복합니다.

지속적으로 최소값을 선택하는 과정을 수행하기 때문에 이 알고리즘을 선택 정렬이라고 부릅니다.

버블 정렬(Bubble Sorting)이란?

버블 정렬은 간단한 정렬 알고리즘입니다. 버블 정렬은 입력 배열의 i번째 요소와 i+1번째 요소를 비교하여 i번째 요소가 i+1 번째 요소보다 클 경우 두 요소의 위치를 바꾸는 작업을 입력 배열의 첫번째부터 마지막까지 반복하비다. 더 이상 교체가 필요없을 때 까지 이 과정을 반복합니다. 일반적으로 버블 정렬보다 삽입 정렬의 복잡도가 더 좋습니다.

성능

최악의 경우 복잡도 : O(n2)

2019/07/22 카테고리 알고리즘

개인적으로 공부한 것을 정리하였습니다.

정렬(Sorting)이란?

정렬이란 어떤 순서에 따라 목록의 요소들을 배열하는 알고리즘입니다. 출력으로는 입력에 대한 경우의 수(순열) 또는 입력을 알고리즘에 따라 다시 배열한 값입니다.

정렬의 분류

비교 횟수에 따른 분류

  • 비교에 기반한 알고리즘들은 최선의 경우 O(nlogn), 최악의 경우 O(n2)의 복잡도를 가집니다.

값을 교환하는 횟수에 따른 분류

  • 값을 교환하는 횟수로 분류됩니다.

메모리 사용에 따른 분류

  • 몇몇 정렬 알고리즘들은 제자리 정렬을 수행하고 정렬을 수행하기 위한 임시적인 확장 공간으로 충분히 무시할 만큼 O(1) 또는 O(logn)의 메모리만을 필요로 합니다.

반복에 따른 분류

  • 정렬 알고리즘은 반복을 사용하거나 사용하지 않거나 두 가지로 분류됩니다. 병합정렬의 경우 두 방법을 모두 사용합니다.
