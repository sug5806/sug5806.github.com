선택 정렬(Selection Sort)이란?

선택 정렬은 제자리 정렬 알고리즘입니다. 선택 정렬은 작은 양의 데이터에 유용합니다. 이 알고리즘은 키는 작고 값은 큰 데이터에 사용됩니다. 키를 기반으로 선택이 이루어지고 치환이 필요하다고 판단되면 치환이 이루어지기 때문입니다.

장점

  • 구현이 쉽습니다.
  • 제자리 정렬(추가 저장 공간이 필요없음) 알고리즘입니다.

단점

  • 데이터의 양에 유연하지 않습니다.

알고리즘

  1. 목록에서 최소값을 찾습니다.
  2. 찾은 최소값을 맨 앞의 값(혹은 현재 위치의 값)과 교체합니다.
  3. 배열의 전체 요소가 정렬될 때까지 이 과정을 반복합니다.

지속적으로 최소값을 선택하는 과정을 수행하기 때문에 이 알고리즘을 선택 정렬이라고 부릅니다.

버블 정렬(Bubble Sorting)이란?

버블 정렬은 간단한 정렬 알고리즘입니다. 버블 정렬은 입력 배열의 i번째 요소와 i+1번째 요소를 비교하여 i번째 요소가 i+1 번째 요소보다 클 경우 두 요소의 위치를 바꾸는 작업을 입력 배열의 첫번째부터 마지막까지 반복하비다. 더 이상 교체가 필요없을 때 까지 이 과정을 반복합니다. 일반적으로 버블 정렬보다 삽입 정렬의 복잡도가 더 좋습니다.

성능

최악의 경우 복잡도 : O(n2)

정렬(Sorting)이란?

정렬이란 어떤 순서에 따라 목록의 요소들을 배열하는 알고리즘입니다. 출력으로는 입력에 대한 경우의 수(순열) 또는 입력을 알고리즘에 따라 다시 배열한 값입니다.

정렬의 분류

비교 횟수에 따른 분류

  • 비교에 기반한 알고리즘들은 최선의 경우 O(nlogn), 최악의 경우 O(n2)의 복잡도를 가집니다.

값을 교환하는 횟수에 따른 분류

  • 값을 교환하는 횟수로 분류됩니다.

메모리 사용에 따른 분류

  • 몇몇 정렬 알고리즘들은 제자리 정렬을 수행하고 정렬을 수행하기 위한 임시적인 확장 공간으로 충분히 무시할 만큼 O(1) 또는 O(logn)의 메모리만을 필요로 합니다.

반복에 따른 분류

  • 정렬 알고리즘은 반복을 사용하거나 사용하지 않거나 두 가지로 분류됩니다. 병합정렬의 경우 두 방법을 모두 사용합니다.

2019-07-18 힙(Heap)

힙(Heap)이란

힙에 기본적인 조건은 노드의 값이 자신의 자식들의 값보다 반드시 크거나 같아야(혹은 작거나 같아야)한다는 것입니다. 이것을 '힙의 특성(heap property)'라고 합니다. 또한 힙은 추가적인 특성들도 가지는데 모든 leaf들은 h>0 일때 (h은 완전 이진 트리에서의 트리의 높이) h 또는 h-1 레벨에 존재합니다. 힙은 완전 이진 트리를 형성해야 합니다.

힙의 종류

  • 최소힙(Min heap): 노드의 값이 자신의 자식 노드의 값보다 작거나 같습니다.

  • 최대힙(Max heap): 노드의 값이 자신의 자식 노드의 값보다 크거나 같습니다.

힙의 추상화(Heap ADT)

힙의 주요 연산들

  • push(data): 힙에 데이터를 넣습니다
  • pop(): 힙에서 데이터를 삭제하고 반환합니다

힙의 보조 연산들

  • is_empty(): 힙이 비어있는지 여부를 반환합니다
  • is_full(): 힙이 가득 차 있는지 여부를 반환합니다
  • top(): 다음번에 삭제할 데이터를 반환합니다.

2019-07-15 우선순위 큐(Priority Queue)

2019/07/15 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

우선순위 큐(Priority Queue)란?

요소들의 집합 내에서 최대 또는 최솟값을 가진 요소를 찾아야 할 때가 있는데, 우선순위 큐 추상 데이터 형(AD이T)이 이런 연산을 제공합니다. 우선순위 큐 추상 데이터형은 Insert, Delete Min 또는 Delete Max와 같은 기능을 제공하는 데이터 구조입니다. 우선순위 큐의 차이점은 큐에 입력되는 순서가 처리되는 순서와 같지 않을 수 있다는 것입니다. 작업 스케줄링이 바로 이런 우선순위 큐를 활용하는 예입니다.

우선순위 큐에서 더 작은 키를 가진 아이템이 높은 우선순위를 가지는 것(가장 작은 요소가 삭제됨)을 오름차순 큐라고 하고 더 높은 키를 가진 아이템이 높은 우선순위를 가지는 것(가장 큰 요소가 삭제됨)을 내림차순 큐라고 부릅니다.

우선 순위 큐의 추상화(Priority Queue ADT)

우선 순위 큐의 주요 연산들

  • Insert(key, value) : 우선 순위 큐에 키와 함께 데이터를 삽입합니다. 삽입된 요소들은 키를 기준으로 정렬됩니다.
  • DeleteMin/DeleteMax: 최소/최대 키를 가진 요소를 삭제하고 삭제된 요소를 반환합니다.
  • GetMinimum/GetMaximum: 최소/최대 키를 가진 요소를 반환합니다.

우선 순위 큐의 보조 연산들

  • Size: 우선순위 큐 내의 요소들의 수를 반환합니다.
  • Heap Sort: 우선 순위(키)에 따라 우선 순위 큐 내의 요소들을 정렬합니다.

개인적으로 공부한 것을 정리하였습니다.

큐(Queue)란?

큐는(연결 리스트와 스택과 같이)데이터를 저장하는데 사용되는 자료 구조입니다. 큐에서는 데이터가 도착하는 순서가 중요합니다. 일반적으로 큐는 시작 부분부터 순차적으로 기다리고 있는 사람이나 사물의 줄입니다.

  • 정의: 큐란 끝에 삽입되고 열의 처음이 삭제되는 순서를 가진 열입니다. 먼저 삽입된 요소가 먼저 삭제됩니다. 따라서 선입선출(First In Fisrt Out) 또는 후입 후출(Last In Last Out)이라 불립니다.

큐에서 수행할 수 있는 연산에도 특별한 이름이 정해져있습니다. 요소가 큐에 삽입되는 것을 인큐(enqueue), 요소가 큐에서 제거되는 것을 디큐(dequeue)라 부릅니다.

큐의 추상 자료형(ADT)

큐의 삽입과 삭제는 FIFO(First In First Out) 방식을 따라야 합니다.

큐의 주요 연산들

  • Enqueue(data): 큐의 끝에 요소를 삽입합니다
  • Dequeue() : 큐의 맨 앞에서 요소를 제거하고 반환합니다.

큐의 보조 연산들

  • Front(): 요소를 제거하지 않고 큐의 제일 앞에 있는 요소를 반환 합니다.
  • QueueSize(): 큐에 저장된 요소의 수를 반환합니다.
  • IsEmptyQueue(): 큐에 요소가 저장되어있는지 여부를 알려줍니다.

스택(Stack)이란?

스택은 데이터를 저장하는데 사용되는(연결 리스트와 유사한) 간단한 자료 구조입니다. 스택은 데이터가 위치하는 순서가 매우 중요합니다. 식당에 쌓여있는 접시들이 좋은 예입니다. 설거지가 끝난 접시가 순서대로 쌓이는 것이 스택과 같습니다(First In Last Out). 이때 제일 처음 설거지를 했던 접시는 제일 마지막에 사용됩니다. 스택에서 데이터 변경은 특별 이름이 정해져 있습니다. 스택에 요소가 삽입되는 것을 구체적으로 push라 하고 스택에서 요소를 제거하는 것을 pop이라 합니다. 비어있는 스택에서 pop을 시도하는 것을 언더플로우(underflow), 꽉 찬 스택에 요소를 삽입하고자 하는 것을 오버플로우(overflow)라고 합니다.

스택의 추상 자료형(ADT)

스택은 아래 연산들로 추상화할 수 있습니다.

스택의 주요 연산

  • push(data) : 데이터를 스택에 삽입합니다
  • pop() : 스택에서 top이 가리키는 데이터를 제거하고 반환합니다.

스택의 보조 연산

  • top() : 스택에 top이 가리키는 요소를 반환합니다.
  • size() : 스택의 저장된 데이터의 수를 반환합니다.
  • is_empty(): 스택이 비어있는지 여부를 반환합니다
  • is_full(): 스택이 꽉 찼는지 여부를 반환합니다

이중 연결 리스트(Double Linked List)

이중 연결 리스트(양방향 연결 리스트)의 장점은 리스트 내의 주어진 노드를 양방향으로 탐색할수 있다는 것입니다. 단일 연결 리스트에 있는 노드는 이전 노드의 포인터를 획득해야 삭제할 수 있습니다. 하지만 이중 연결 리스트에서는 이전 노드의 주소를 몰라도 삭제할 수 있습니다.

연결 리스트란?

연결 리스트(Linked list)는 데이터 집합을 저장할 때 사용되는 자료 구조입니다. 연결 리스트는 다음과 같은 속성을 가집니다.

  • 연속된 요소들은 포인터로 연결됩니다.
  • 마지막 요소는 NULL을 가리킵니다.
  • 프로그램 실행 중에 크기가 늘거나 줄 수 있습니다.
  • 필요한 만큼 만들 수 있습니다.(시스템 메모리를 다 쓸 때까지).
  • 메모리 공간을 낭비하지 않습니다.
기본 구조

연결 리스트의 추상 데이터 형(ADT)

연결 리스트의 추상 함수는 다음과 같습니다.

  • 연결 리스트 주요 함수
    • Insert : 리스트에 요소를 추가합니다.
    • Delete : 리스트의 지정된 위치의 요소를 제거하고 반환합니다.
  • 연결 리스트의 보조 함수
    • Delete List : 리스트의 모든 요소들을 제거합니다.
    • Count : 리스트 내 모든 요소의 수를 반환합니다.

연결 리스트의 장점

연결 리스트는 장단점을 가지고 있습니다. 연결 리스트의 장점은 일정 시간(고정된 시간) 내에 확장될 수 있다는 것입니다. 배열의 경우 생성할 때 요소들의 수 만큼 메모리를 할당해야 합니다. 배열에 새로운 요소를 추가하기 위해서는 새로운 배열을 생성하고 기존의 배열의 요소들을 새로운 배열로 복사해야 하기 때문에 시간을 많이 소모 합니다. 연결리스트는 이러한 문제 없이 쉽게 새로운 요소를 추가할 수 있습니다.

연결 리스트의 단점

연결 리스트는 리스트 내 요소에 접근하는데 최악의 경우 O(n)을 가집니다. 비록 연결 리스트의 동적인 저장 공간의 할당이 큰 장점이라 하더라도 데이터의 정렬과 탐색의 오버헤드는 다른 상황을 만들 수 있습니다.
