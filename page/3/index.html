2019-07-11 큐(Queue)

2019/07/11 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

큐(Queue)란?

큐는(연결 리스트와 스택과 같이)데이터를 저장하는데 사용되는 자료 구조입니다. 큐에서는 데이터가 도착하는 순서가 중요합니다. 일반적으로 큐는 시작 부분부터 순차적으로 기다리고 있는 사람이나 사물의 줄입니다.

  • 정의: 큐란 끝에 삽입되고 열의 처음이 삭제되는 순서를 가진 열입니다. 먼저 삽입된 요소가 먼저 삭제됩니다. 따라서 선입선출(First In Fisrt Out) 또는 후입 후출(Last In Last Out)이라 불립니다.

큐에서 수행할 수 있는 연산에도 특별한 이름이 정해져있습니다. 요소가 큐에 삽입되는 것을 인큐(enqueue), 요소가 큐에서 제거되는 것을 디큐(dequeue)라 부릅니다.

큐의 추상 자료형(ADT)

큐의 삽입과 삭제는 FIFO(First In First Out) 방식을 따라야 합니다.

큐의 주요 연산들

  • Enqueue(data): 큐의 끝에 요소를 삽입합니다
  • Dequeue() : 큐의 맨 앞에서 요소를 제거하고 반환합니다.

큐의 보조 연산들

  • Front(): 요소를 제거하지 않고 큐의 제일 앞에 있는 요소를 반환 합니다.
  • QueueSize(): 큐에 저장된 요소의 수를 반환합니다.
  • IsEmptyQueue(): 큐에 요소가 저장되어있는지 여부를 알려줍니다.

2019-07-08 스택(Stack)

2019/07/08 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

스택(Stack)이란?

스택은 데이터를 저장하는데 사용되는(연결 리스트와 유사한) 간단한 자료 구조입니다. 스택은 데이터가 위치하는 순서가 매우 중요합니다. 식당에 쌓여있는 접시들이 좋은 예입니다. 설거지가 끝난 접시가 순서대로 쌓이는 것이 스택과 같습니다(First In Last Out). 이때 제일 처음 설거지를 했던 접시는 제일 마지막에 사용됩니다. 스택에서 데이터 변경은 특별 이름이 정해져 있습니다. 스택에 요소가 삽입되는 것을 구체적으로 push라 하고 스택에서 요소를 제거하는 것을 pop이라 합니다. 비어있는 스택에서 pop을 시도하는 것을 언더플로우(underflow), 꽉 찬 스택에 요소를 삽입하고자 하는 것을 오버플로우(overflow)라고 합니다.

스택의 추상 자료형(ADT)

스택은 아래 연산들로 추상화할 수 있습니다.

스택의 주요 연산

  • push(data) : 데이터를 스택에 삽입합니다
  • pop() : 스택에서 top이 가리키는 데이터를 제거하고 반환합니다.

스택의 보조 연산

  • top() : 스택에 top이 가리키는 요소를 반환합니다.
  • size() : 스택의 저장된 데이터의 수를 반환합니다.
  • is_empty(): 스택이 비어있는지 여부를 반환합니다
  • is_full(): 스택이 꽉 찼는지 여부를 반환합니다

2019-07-01 연결 리스트(Linked List)1

2019/07/01 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

연결 리스트란?

연결 리스트(Linked list)는 데이터 집합을 저장할 때 사용되는 자료 구조입니다. 연결 리스트는 다음과 같은 속성을 가집니다.

  • 연속된 요소들은 포인터로 연결됩니다.
  • 마지막 요소는 NULL을 가리킵니다.
  • 프로그램 실행 중에 크기가 늘거나 줄 수 있습니다.
  • 필요한 만큼 만들 수 있습니다.(시스템 메모리를 다 쓸 때까지).
  • 메모리 공간을 낭비하지 않습니다.
기본 구조

연결 리스트의 추상 데이터 형(ADT)

연결 리스트의 추상 함수는 다음과 같습니다.

  • 연결 리스트 주요 함수
    • Insert : 리스트에 요소를 추가합니다.
    • Delete : 리스트의 지정된 위치의 요소를 제거하고 반환합니다.
  • 연결 리스트의 보조 함수
    • Delete List : 리스트의 모든 요소들을 제거합니다.
    • Count : 리스트 내 모든 요소의 수를 반환합니다.

연결 리스트의 장점

연결 리스트는 장단점을 가지고 있습니다. 연결 리스트의 장점은 일정 시간(고정된 시간) 내에 확장될 수 있다는 것입니다. 배열의 경우 생성할 때 요소들의 수 만큼 메모리를 할당해야 합니다. 배열에 새로운 요소를 추가하기 위해서는 새로운 배열을 생성하고 기존의 배열의 요소들을 새로운 배열로 복사해야 하기 때문에 시간을 많이 소모 합니다. 연결리스트는 이러한 문제 없이 쉽게 새로운 요소를 추가할 수 있습니다.

연결 리스트의 단점

연결 리스트는 리스트 내 요소에 접근하는데 최악의 경우 O(n)을 가집니다. 비록 연결 리스트의 동적인 저장 공간의 할당이 큰 장점이라 하더라도 데이터의 정렬과 탐색의 오버헤드는 다른 상황을 만들 수 있습니다.

재귀(Recursion)란?

자기 자신을 반복해서 호출하는 모든 함수를 재귀적(recursive)라 합니다. 재귀적 방법은 보다 작은 문제로 작업하기 위해서 자기 자신의 복사본을 호출함으로써 문제를 해결합니다. 이것을 재귀 단계라 합니다. 재귀 단계에서는 더 많은 동일한 재귀 호출이 발생할 수 있습니다. 재귀 호출은 종료되도록 하는 것이 중요합니다. 자기 자신을 호출할 때마다 원래 문제의 범위는 조금씩 줄어듭니다. 이 일련의 작은 문제들은 결국 base case에 도달합니다 [1]

왜 재귀방식을 사용할까?

일반적으로 재귀 코드는 반복 코드보다 짧고 쉽습니다. 재귀는 보다 작은 하위 작업으로 정의될 수 있는 작업에 가장 유용합니다. 예를 들어 정렬, 검색 그리고 탐색 문제는 간단한 재귀 방식을 가지고 해결합니다.

재귀함수의 예시

팩토리얼(factorial)을 예시로 들어보겠습니다.

2019-06-28 자료 구조란?

2019/06/28 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

자료구조(Data Structure)

자료구조는 컴퓨터 내에서 유용하게 사용되도록 데이터를 저장하고 구조화하는 특정한 방법을 말합니다. 즉, 데이터를 저장하고 구조화하는 특별한 형태입니다. 대표적인 자료구조는 배열, 연결 리스트, 스택, 큐, 트리, 그래프 등이 있습니다.

자료구조는 구성요소들을 어떻게 구조화하였는가에 따라 두 가지 유형으로 분류됩니다.

  • 선형 자료구조(Linear data structures) : 선형 자료구조의 구성 요소들은 순차적으로 접근되지만, 모든 요소들이 반드시 순차적으로 저장되지는 않습니다. (Ex.연결 리스트, 스택, 큐)
  • 비선형 자료 구조(Non-linear data structures): 자료 구조의 구성 요소들이 비순차적인 순서로 저장되고 접근됩니다. (Ex. 트리, 그래프)

추상 자료형(Abstract Data Types - ADTs)

자료 구조와 연산을 결합하여 추상 자료형이라고 부릅니다. 추상 자료형은 두 부분으로 구성됩니다.

  • 데이터 선언
  • 연산자 선언
    일반적으로 사용되는 추상 자료형은 연결 리스트, 스택, 큐, 우선 순위 큐, 이진 트리, 딕셔너리, 분리 집합, 해시 테이블, 그래프 등이 있습니다. 예를 들어 스택은 자료 구조에 데이터를 저장할 때 LIFO(Last-In-First-Out) 메커니즘을 사용합니다. 스택에 마지막으로 삽입된 요소는 제일 먼저 빠져나와 제거됩니다. 스택의 일반적인 연산은 스택 생성, 스택에 삽입하는 push, 제거하는 pop, 최상위 데이터를 찾는 top 등이 있습니다.

선형 자료구조와 비선형 자료구조

자료구조는 크게 선형 자료구조와 비선형 자료구조로 나뉘어 집니다.

  • 선형 자료구조란 하나의 데이터 뒤에 하나의 데이터가 존재하는 것입니다.

  • 데이터와 데이터간의 관계가 1:1 관계 입니다

  • 대표적으로는 배열과 리스트가 있습니다

  • 비선형 자료구조란 하나의 데이터 뒤에 여러개의 데이터가 존재할 수 있습니다.
  • 데이터간의 관계가 1:N 또는 N:M의 관계가 있습니다
  • 대표적인 비선형 자료구조는 트리와 그래프가 있습니다.
