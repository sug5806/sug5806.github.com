2019-09-04 운영체제1

2019/09/04 카테고리 운영체제

Operating System Concepts의 요약본 입니다.

1. 소개

  • 운영체제란?
    • 컴퓨터 하드웨어를 관리하는 프로그램
    • 응용프로그램의 토대를 제공해주는 프로그램
    • 컴퓨터의 사용자와 컴퓨터 하드웨어 사이에 중간 매개체 역할을 해주는 프로그램

1.1 운영체제란?

  • 컴퓨터 시스템의 구성요소
    • 하드웨어
    • 운영체제
    • 응용프로그램
    • 사용자
  • 운영체제 범위에 대한 정의
    • 정의 1. 운영체제를 구입하였을 때 포함되어 있는 모든 것
    • 정의 2. 항상 수행되고 있는 유일한 프로그램만 운영체제이다. 보통 다른 말로 커널(kernel)이라 한다.

1.2 메인프레임 시스템

  • 한 사용자가 하나의 프로그램만을 수행하면 운행체제의 일은 그만큼 쉽다. 반면에 한 사용자가 여러 프로그램을 동시에 수행하거나 여러 사용자가 동시에 사용하는 시스템에서는 운영체제의 일은 간단하지 않다.

1.2.1 일괄처리 시스템

  • 한번에 하나의 프로그램만이 수행되므로 운영체제의 주 임무는 하나의 작업에서 다음 작업으로 제어를 자동적으로 옮기는 것이었다.

  • 운영체제는 항상 메모리에 상주하였다.

  • 처리속도를 향상하기 위해 유사한 요구를 필요로 하는 작업을 함께 모아서 일괄처리하였다.

  • 문제점

    • 기계적 입출력 장치가 전자적 장치의 속도보다 상대적으로 느려 CPU가 종종 쉬는 경우가 많았다.

  • 직접 접근(direct access)이 가능한 디스크의 도입

    • 작업 스케줄링(jop scheduling) 과 다중 프로그래밍(multiprogramming) 이 가능하게됨.

1.2.2 다중프로그램 시스템

  • 다중 프로그래밍이란 여러 개의 프로그램을 동시에 메모리에 적재하여 하나의 프로그램이 대기 상태가되면 그 동안 다른 프로그램을 실행하는 방식을 말한다.

  • 장점

    • 입출력과 프로그램의 실행을 병행으로 수행할 수 있어 CPU의 사용 효율이 증가함

  • 다중 프로그래밍에서 운영체제는 사용자를 대신하여 의사결정을 하여야 한다.

    • 디스크에 있는 작업 저장소에서 작업을 선택하여 메모리로 옮기는 것을 작업 스케줄링이라 한다.
    • 실행중인 작업이 대기 상태가 되었을 때 메모리에 있는 작업 중 하나를 선택하여 CPU에 항당하는 것은 CPU 스케줄링이라 한다.

  • 여러 프로그램이 동시에 메모리에 상주하므로 메모리 관리가 복잡하다.

1.2.3 시분할 시스템

  • 다중 프로그래밍을 제공하는 일괄처리 시스템은 CPU의 사용 효율은 높였지만 사용자와 컴퓨터 간에 상호작용은 제공하지 못하였다.

  • 시분할(time sharing)또는 멀티태스킹은 다중 프로그래밍과 달리 정해진 시간이 되면 무조건 다음 순서의 작업을 실행하는 방식으로, 교대하는 시간이 매우 짧아 프로그램이 실행되는 동안 사용자는 컴퓨터와 상호작용이 가능하다.

  • 이처럼 사용자와 시스템 간에 직접 상호작용이 가능한 시스템을 대화식 컴퓨터 시스템이라 한다. 이런 시스템에서는 응답시간(response time)이 짧아야 한다.

  • 시분할 시스템은 여러 사용자가 동시에 컴퓨터를 사용할 수 있게 해준다.

  • 메모리에 적재되어 실행중인 프로그램을 프로세스(process) 라 한다.

  • 많은 사용자의 프로그램을 동시에 수행하기 위해서는 주기억장치의 용량으로는 부족하다. 이 문제를 해결하기 위해 디스크를 주기억장치의 보조 저장장치로 활용하며, 현재 가장 널리 사용되는 기법은 가상 메모리(virtual memory)이다.

그래프

현실에서 많은 문제들은 객체와 객체 사이의 관계로 표현됩니다. 항공기 운항 경로를 예로 들면 "서울에서 뉴욕까지 가는 가장 빠른 노선은 어떻게 됩니까?"와 같은 질문이 있습니다. 이 질문에 답하기위해 도시 간의 관계에 대한 정보가 필요합니다. 이런 문제를 풀기 위해 사용되는 자료 구조가 그래프(Graph)입니다.

용어 설명

그래프(Graph): 그래프는 정점(vertices)라고 불리는 노드들의 집합 V와 간선(edge)이라고 불리는 정점의 쌍들의 집합 E를 사용하여(V,E)로 나타냅니다.

  • 정점과 간선은 위치 및 저장 요소입니다.
  • 그래프에서 사용하는 정의:
    • 방향을 가진 간선(Directed edge):
      • 순서를 가진 정점의 쌍(u,v)으로
      • 첫 번째 정점 u는 출발점을 의미합니다.
      • 두 번째 정점 v는 도착점을 의미합니다.
      • 예 : 편도 운행

  • 방향을 가지지 않은 간선(Undirected edge)
    • 순서를 가지지 않는 정점의 쌍 (u,v)
    • 예 : 기찻길

  • 방향을 가지는 그래프(Directed graph)
    • 그래프 내의 모든 간선들이 방향을 가집니다.
    • 예 - 네트워크의 라우트

  • 방향을 가지지 않는 그래프(Undirected graph)
    • 그래프 내의 모든 간선이 방향을 가지지 않습니다.
    • 예 : 비행노선

개인적으로 공부한 것을 정리하였습니다.

이진 트리(Binary Tree)란 무엇인가?

모든 노드들이 최대 2개의 자식을 가질 때 이를 이진 트리라고 합니다. root에서부터 오른쪽과 왼쪽의 하위 트리들로 분리하여 아래와 같이 나타낼 수 있습니다

이진트리의 유형

포화 이진트리(Full Binary Tree)

모든 leaf 노드들이 동일한 레벨인 트리를 포화 이진트리라 합니다.

완전 이진 트리(Complete Binary Tree)

마지막 레벨을 제외한 각 레벨이 빠짐없이 채워져 있고, 마지막 레벨은 중간에 빈칸 없이 왼쪽부터 노드들로 채워져 있어야 합니다.

트리의 연산

기본 연산

  • 트리에 요소를 삽입
  • 트리에서 요소를 제거
  • 요소 검색(Searching)
  • 트리 운행(Traversing)

보조 연산

  • 트리의 크기 찾기
  • 트리의 높이 찾기
  • 최대 합을 가진 레벨 찾기
  • 주어진 둘 이상의 노드에 대한 최소 공통 조상(LCA) 찾기

이진 트리 운행(Traversals)

트리를 다루기 위해서는 트리를 운행하기 위한 메커니즘 필요한데 이것이 여기서 다룰 내용입니다. 트리의 모든 노드들은 방문하는 과정을 트리 운행이라 합니다. 각 노드들은 한 번만 처리되겠지만, 여러 번 방문을 받을 수 있습니다.

운행 방법(Traversal Possibilities)

운행 방법을 아래와 같이 3가지로 나타낼 수 있습니다.

  • 전위 운행법(Preorder)
  • 중위 운행법(Inorder)
  • 후위 운행법(Postorder)

여기에 순서를 따르지 않는 다른 운행법을 추가합니다.

  • 레벨 순서 운행법(Level Order Traversal): 이 함수는 너비 우선 탐색에서 영향을 받았습니다(그래프 알고리즘의 BFS).

전위 운행법(Preorder)

전위 운행법에서는 각 노드의 자신의 좌, 우측 하위 트리보다 먼저 처리됩니다.

  • root를 방문합니다.
  • 다시 전위 운행법으로 왼쪽 하위 트리를 운행합니다.
  • 전위 운행법으로 오른쪽 하위 트리를 운행합니다.

위의 트리에서는 1, 2, 4, 5, 3, 6, 7 순서로 처리됩니다.

중위 운행법(Inorder)

중위 운행법에서 root는 오른쪽 하위 트리와 왼쪽 하위 트리 사이에 방문하게 됩니다. 중위 운행법은 다음과 같이 정의됩니다.

  • 왼쪽 하위트리를 탐색합니다.
  • root를 처리합니다.
  • 오른쪽 하위 트리를 중위 운행법으로 탐색합니다.

위의 트리에서는 4, 2, 5, 1, 6, 3, 7 순서로 처리됩니다.

후위 운행법(Postorder)

후위 운행에서 root는 양쪽 하위 트리가 처리된 이후 처리됩니다. 후위 운행법은 다음과 같이 정의합니다.

  • 왼쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • 오른쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • root를 처리합니다.

앞의 예제 트리의 노드들은 4, 5, 2, 6, 7, 3, 1 순서로 처리됩니다.

트리란 무엇인가?

트리란 연결 리스트와 유사한 자료 구조지만 연결 리스트가 단순히 다음 노드를 가리키는 선형이라면 트리느 각 노드가 여러 개의 노드를 가리키는 비선형입니다. 트리는 비선형 자료 구조의 대표적인 예이면서, 그래픽 형태로 계층적 구조의 특성을 표현하는 방법입니다. 트리의 추상 자료형(ADT)에서는 요소들의 순서가 중요하지 않습니다. 만일 순서가 중요하다면 연결 리스트, 스택, 큐와 같은 선형 자료 구조를 사용해야 합니다.

용어 정의

  • 트리의 root는 부모를 가지지 않습니다. 트리에는 최대 하나의 root노드가 있을 수 있습니다.
  • edge는 부모와 자식을 잇는 연결 선입니다(그림에서의 모든 연결 선).
  • 자식이 없는 노드를 leaf 노드라고 부릅니다(E, J, K, H, I).
  • 같은 부모의 자식을 siblings라고 부릅니다(E, J, K, H, I).
  • 노드의 depth는 root에서 해당 노드까지 경로(path)의 길이입니다.(G의 depth는 2(A-C-G)입니다)
  • 트리의 height는 트리 내 모든 노드들 중 높이가 가장 큰 값이고 트리의 depth는 모든 노드들 중에서 depth가 가장 큰 값입니다.
  • 지정된 depth에 위치한 모든 노드들을 트리의 레벨(level)이라 부릅니다. (예에서 B, C 그리고 D는 같은 레벨입니다). Root 노드의 레벨은 0입니다.

개인적으로 공부한 것을 정리하였습니다.

UDP(User Datagram Protocol)

UDP 프로토콜은 네트워크에 있는 두 개의 응용 프로그램 간의 보장할 수 없는 통신을 제공합니다. UDP는 TCP같은 연결기반이 아닙니다. 한 어플리케이션에서 다른 어플리케이션으로 데이터그램(Datagram)이라고 부르는 독립적인 데이터 패킷을 전송합니다. 데이터그램 전송은 우편 서비스를 통해 편지를 보내는 것과 매우 유사합니다. 배달 순서는 중요하지 않고 각 메시지는 다른 메시지와 독립적입니다.

UDP 프로토콜은 데이터의 도착을 보장하지 않고 프로토콜 내 내장된 전달을 보증하거나 에러를 탐지하여 에러를 교정하는 메커니즘이 없기 때문에 TCP보다 신뢰할 수 없습니다.

FTP, SMTP 그리고 TCP 같은 어플리케이션 프로토콜은 클라이언트와 서버 프로그램 간의 신뢰할 수 있는 스트림 기반 통신을 제공하기 위해 TCP를 사용합니다.
