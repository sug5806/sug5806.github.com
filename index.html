이진 트리 운행(Traversals)

트리를 다루기 위해서는 트리를 운행하기 위한 메커니즘 필요한데 이것이 여기서 다룰 내용입니다. 트리의 모든 노드들은 방문하는 과정을 트리 운행이라 합니다. 각 노드들은 한 번만 처리되겠지만, 여러 번 방문을 받을 수 있습니다.

운행 방법(Traversal Possibilities)

운행 방법을 아래와 같이 3가지로 나타낼 수 있습니다.

  • 전위 운행법(Preorder)
  • 중위 운행법(Inorder)
  • 후위 운행법(Postorder)

여기에 순서를 따르지 않는 다른 운행법을 추가합니다.

  • 레벨 순서 운행법(Level Order Traversal): 이 함수는 너비 우선 탐색에서 영향을 받았습니다(그래프 알고리즘의 BFS).

전위 운행법(Preorder)

전위 운행법에서는 각 노드의 자신의 좌, 우측 하위 트리보다 먼저 처리됩니다.

  • root를 방문합니다.
  • 다시 전위 운행법으로 왼쪽 하위 트리를 운행합니다.
  • 전위 운행법으로 오른쪽 하위 트리를 운행합니다.

위의 트리에서는 1, 2, 4, 5, 3, 6, 7 순서로 처리됩니다.

중위 운행법(Inorder)

중위 운행법에서 root는 오른쪽 하위 트리와 왼쪽 하위 트리 사이에 방문하게 됩니다. 중위 운행법은 다음과 같이 정의됩니다.

  • 왼쪽 하위트리를 탐색합니다.
  • root를 처리합니다.
  • 오른쪽 하위 트리를 중위 운행법으로 탐색합니다.

위의 트리에서는 4, 2, 5, 1, 6, 3, 7 순서로 처리됩니다.

후위 운행법(Postorder)

후위 운행에서 root는 양쪽 하위 트리가 처리된 이후 처리됩니다. 후위 운행법은 다음과 같이 정의합니다.

  • 왼쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • 오른쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • root를 처리합니다.

앞의 예제 트리의 노드들은 4, 5, 2, 6, 7, 3, 1 순서로 처리됩니다.

이진 트리(Binary Tree)란 무엇인가?

모든 노드들이 최대 2개의 자식을 가질 때 이를 이진 트리라고 합니다. root에서부터 오른쪽과 왼쪽의 하위 트리들로 분리하여 아래와 같이 나타낼 수 있습니다

이진트리의 유형

포화 이진트리(Full Binary Tree)

모든 leaf 노드들이 동일한 레벨인 트리를 포화 이진트리라 합니다.

완전 이진 트리(Complete Binary Tree)

마지막 레벨을 제외한 각 레벨이 빠짐없이 채워져 있고, 마지막 레벨은 중간에 빈칸 없이 왼쪽부터 노드들로 채워져 있어야 합니다.

트리의 연산

기본 연산

  • 트리에 요소를 삽입
  • 트리에서 요소를 제거
  • 요소 검색(Searching)
  • 트리 운행(Traversing)

보조 연산

  • 트리의 크기 찾기
  • 트리의 높이 찾기
  • 최대 합을 가진 레벨 찾기
  • 주어진 둘 이상의 노드에 대한 최소 공통 조상(LCA) 찾기

개인적으로 공부한 것을 정리하였습니다.

트리란 무엇인가?

트리란 연결 리스트와 유사한 자료 구조지만 연결 리스트가 단순히 다음 노드를 가리키는 선형이라면 트리느 각 노드가 여러 개의 노드를 가리키는 비선형입니다. 트리는 비선형 자료 구조의 대표적인 예이면서, 그래픽 형태로 계층적 구조의 특성을 표현하는 방법입니다. 트리의 추상 자료형(ADT)에서는 요소들의 순서가 중요하지 않습니다. 만일 순서가 중요하다면 연결 리스트, 스택, 큐와 같은 선형 자료 구조를 사용해야 합니다.

용어 정의

  • 트리의 root는 부모를 가지지 않습니다. 트리에는 최대 하나의 root노드가 있을 수 있습니다.
  • edge는 부모와 자식을 잇는 연결 선입니다(그림에서의 모든 연결 선).
  • 자식이 없는 노드를 leaf 노드라고 부릅니다(E, J, K, H, I).
  • 같은 부모의 자식을 siblings라고 부릅니다(E, J, K, H, I).
  • 노드의 depth는 root에서 해당 노드까지 경로(path)의 길이입니다.(G의 depth는 2(A-C-G)입니다)
  • 트리의 height는 트리 내 모든 노드들 중 높이가 가장 큰 값이고 트리의 depth는 모든 노드들 중에서 depth가 가장 큰 값입니다.
  • 지정된 depth에 위치한 모든 노드들을 트리의 레벨(level)이라 부릅니다. (예에서 B, C 그리고 D는 같은 레벨입니다). Root 노드의 레벨은 0입니다.

UDP(User Datagram Protocol)

UDP 프로토콜은 네트워크에 있는 두 개의 응용 프로그램 간의 보장할 수 없는 통신을 제공합니다. UDP는 TCP같은 연결기반이 아닙니다. 한 어플리케이션에서 다른 어플리케이션으로 데이터그램(Datagram)이라고 부르는 독립적인 데이터 패킷을 전송합니다. 데이터그램 전송은 우편 서비스를 통해 편지를 보내는 것과 매우 유사합니다. 배달 순서는 중요하지 않고 각 메시지는 다른 메시지와 독립적입니다.

UDP 프로토콜은 데이터의 도착을 보장하지 않고 프로토콜 내 내장된 전달을 보증하거나 에러를 탐지하여 에러를 교정하는 메커니즘이 없기 때문에 TCP보다 신뢰할 수 없습니다.

FTP, SMTP 그리고 TCP 같은 어플리케이션 프로토콜은 클라이언트와 서버 프로그램 간의 신뢰할 수 있는 스트림 기반 통신을 제공하기 위해 TCP를 사용합니다.

TCP

TCP는 신뢰성 있는 통신이 필요로 한 애플리케이션을 위해 점대점 채널을 제공합니다. 하이퍼텍스트 전송 프로토콜(HTTP), 파일 전송 프로토콜(FTP) 및 텔넷은 신뢰할 수 있는 통신 채널이 필요로 한 애플리케이션입니다. 데이터가 네트워크를 통해 전송 및 수신되는 순서는 이들 애플리케이션의 성공에 매우 중요합니다. HTTP로 URL에서 데이터를 읽을 때 데이터는 보내진 순서대로 수신되어야 합니다. 그렇지 않으면 뒤죽박죽된 HTML 파일, 손상된 압출 파일 또는 다른 잘못된 정보를 받게 됩니다.

  • Application: HTTP, FTP, telnet, SMTP 등
  • Transport: TCP, UCP 등
  • Network: IP 등
  • Link: Device driver 등

전송 프로토콜(Transport protocols)은 하나의 포트에서 다른 포트로 정보를 전달해 애플리케이션 프로그램 간을 통신할 수 있게 하기 위해 사용됩니다. 이들은 연결지향 또는 비연결지향 통신 방식 중 하나를 사용합니다. TCP는 연결 지향형 프로토콜이며 UDP는 비연결 지향형 전송 프로토콜입니다.

연결 지향형 프로토콜 TCP는 출발지 포트/IP 주소와 도착지 포트/IP 주소 간의 통신 연결을 수립합니다. 연결이 종료되고 링크가 끊어질 때까지 포트들은 이 링크를 통해 서로 연결됩니다. 연결 지향 프로토콜의 한 예가 전화 통화입니다. 전화 연결이 수립되어 통신이 이루어진 다음 연결이 종료됩니다.

MVC 디자인 패턴이란?

모델-뷰-컨트롤러(MVC)는 구조적 패턴입니다. MVC 디자인 패턴에서, 응용 프로그램은 모델(Model), 뷰(View), 컨트롤러(Controller)로 알려진 컴포넌트로 나뉘어 집니다. 이 패턴의 목표는 응용 프로그램의 입력(Controller), 비즈니스 처리 로직(Model) 그리고 출력 로직(View)로 나누는 것입니다.

  • 컨트롤러는 모델과 뷰에 사용자 입력과 연관됩니다.
  • 모델은 저장소에서 표현될 데이터를 불러옵니다.
  • 뷰는 불러온 데이터를 어떻게 사용자에게 표현할 것인가를 다룹니다

컨트롤러는 사용자와 처리(Model) 그리고 서식(View)로직 사이의 중간자로 생각할 수 있습니다. 모든 어플리케이션에 대한 모든 사용자 요청 또는 입력의 진입 지점입니다. 컨트롤은 사용자의 입력을 받아들여 그것을 분석하고 어떤 모델과 뷰를 호출할 것인지를 결정합니다. 이에 따라 요청된 정보를 사용자에게 제공하기 위해 선택된 모델과 뷰를 호출합니다.

모델은 응용프로그램의 비즈니스 처리 로직을 나타냅니다. 이것은 응용프로그램 로직을 캡슐화한 것입니다. 모델은 또한 데이터베이스와 연동하고 삽입, 갱신 그리고 삭제 연산을 수행할 책임이 있습니다.

모든 모델은 호출된 후 컨트롤에 특정 종류의 데이터를 제공합니다. 더 나아가 단일 모델이 호출된 모델의 메서드에 기반해 동일한 종류의 데이터에 대한 여러 가지 다른 변형을 반활할 수 있습니다. 정확히 컨트롤에 반환되는 것은 모델의 메서드에 전달되는 인자들로 제어할 수 있습니다.

하노이의 탑이란?

하노이의 탑은 퍼즐의 일종입니다. 세 개의 기둥과 이 기둥에 꽂을 수 있는 크기가 다양한 원판들이 있고, 퍼즐을 시작하기 전에는 한 기둥에 원판들이 작은 것이 위에 있도록 순서대로 쌓여 있습니다.
게임의 목적은 다음 두 가지 조건을 만족하게 하면서, 한 기둥에 꽂힌 원판들을 그 순서 그대로 다른 기둥으로 옮겨서 다시 쌓는 것입니다.

  1. 한 번에 하나의 원판만 옮길 수 있습니다.
  2. 큰 원판이 작은 원판 위에 있어서는 안 됩니다.

검색이란?

컴퓨터 공학에서 말하는 검색이란 어떤 요소들의 집합에서 특정 속성을 가지는 요소를 찾는 과정을 말합니다. 요소들은 데이터베이스의 레코드, 배열의 요소, 파일의 텍스트, 트리의 노드, 그래프의 정점 혹은 간선이거나 또는 다른 검색 공간의 기타 요소들일 수 있습니다.

검색의 종류

  • 불규칙 선형 검색
  • 정렬/규칙 선형검색
  • 이진 검색

불규칙 선형 검색

배열의 요소들이 정렬되어 있지 않으면 어떤 요소를 찾고 싶다면 주어신 배열에 해당 요소가 있는지를 모든 요소에 대해 검사해야 합니다.

시간 복잡도 : 최악의 경우 배열의 모든 요소를 검사해야 하기 때문에 O(n)

정렬/규칙 선형 검색

배열이 정렬되었을 경우 해당 배열이 찾고자 하는 요소를 가지고 있는지 여부를 알기 위해 배열의 모든 요소를 조사할 필요가 없습니다.

시간 복잡도 : 최악의 경우 배열의 모든 요소를 검사해야 하기 때문에 O(n)

이진 검색

사전에서 어떤 단어를 찾는다고 생각해보면, 일반적으로 인접한 페이지를 펴서 거기서부터 찾기 시작합니다. 그리고 우리가 찾는 단어를 찾으면 검색은 끝이 납니다. 펼친 페이지가 찾고자 하는 단어 이전이면, 나머지 페이지들을 가지고 처음 과정을 반복하고 그렇지 않으면 앞 페이지에 처음 과정을 반복합니다.

퀵 정렬(Quick) Sort)이란?

퀵 정렬 또한 분할 정복 알고리즘 기술의 좋은 예입니다. 퀵 정렬은 분할 교한 정렬이라고도 하며 요소들을 정렬하기 위해 재귀 호출을 사용합니다.

알고리즘

재귀 호출 알고리즘은 4단계로 구성됩니다.

  1. 배열에 정렬될 요소가 하나 이하일 때 return 합니다.
  2. 피봇(pivot) 지점 역할을 하는 요소를 배열에서 선택합니다
  3. 배열을 두개의 부분으로 분할합니다. 하나는 피봇 값 보다 큰 요소들이고 다른 하나는 피봇 값 보다 작은 요소들이 됩니다.
  4. 원래 배열을 양분한 두 배열에 대해 재귀적으로 알고리즘을 반복합니다.
