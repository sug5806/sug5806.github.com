2019-04-07 TCP/IP 4계층

2019/04/07 카테고리 네트워크

OSI 7 Layer 와 TCP/IP

  • 응용 계층(Application Layer) = Application
  • 표현 계층(Presentation Layer)
  • 세션 계층(Session Layer)
  • 전송 계층(Transport Layer) = Transport
  • 네트워크 계층(Network Layer) = Internet
  • 데이터 링크 계층(Data link Layer) = Network Interface
  • 물리 계층(Physical Layer)

TCP/IP에서 알아야할 프로토콜

응용 계층(Application Layer) = FTP, HTTP, SSH
전송 계층(Transport Layer) = TCP, UDP
네트워크 계층(Network Layer) = IP
데이터 링크 계층(Data link Layer) = Ethernet

TCP/IP 4계층

네트워크 전송 시 데이터 표준을 정리한것이 OSI 7 Layer이고 이 것을 바탕으로 실제 사용하는 인터넷 표준이 TCP/IP 4계층이다.

4 계층 : Application (OSI 7Layer의 5, 6, 7계층 담당)

  • 주요 프로토콜
    • FTP
    • Telnet : 보안에 너무 취약하여 현재는 안쓰임
    • DNS(Domain name System) : IP와 Name(Domain)을 이어주는 역할
      • 사용자가 보내면 request를 받아야하는데 이 역할을 하는게 DNS server
    • HTTP
      • http://www.naver.com///~/
      • schema/host/ /디렉토리들/
      • 1.1 부터는 clnt랑 server랑 한번 연결하면 필요없을때까지 연결을 끊지 않음

3계층 : Transport (OSI 7Layer의 4계층 담당)

port(socket에 부여된 번호, port는 프로세스안에 있음)번호를 관리함

  1. TCP(E-mail, 채팅)
    • 신뢰도 높음
      • timeout이 났을 경우 자동으로 재전송
    • 연결 지향적
      • 상대방에게 패킷을 보내고 받았는지 체크(ACK)
    • TCB
      • 서로 연결이 되면 TCB에 등록을 해놓음
      • 소켓쌍 - (source IP, source port), (dest IP, dest port)
      • send buffer, receive buffer에 대한 포인터
        • send와 receive -> 커널 영역
    • sliding window
      • receive window : 내가 받을 수 있는 패킷의 양
    • 3 Way handShaking
      • byte stream에 첫번째 부여하는 숫자를 ISN(initial sequence number)이라 한다
      • 그 숫자를 sequence라고 함
        1. client가 server에 SYC와 자기가 보내고자한 바이트의 seq를 같이 보냄
        1. server역시 SYC + ACK(상대방이 보낸 seq에 +1을 해서)를 보내 자신의 send buffer와 client의 receive buffer와 동기화 시킴
    • TearDown
        1. client가 FIN를 보냄
        1. Server는 ACK를 보냄 -> clinet는 send는 못하지만 receive는 가능, server가 남은 데이터를 send함
        1. 남은 데이터를 보내고 난뒤 server가 FIN를 보냄 (Time Out만큼 기다림)
        1. client가 server의 FIN을 받았으므로 ACK을
  2. UDP(스트리밍 서비스, 게임)
    • 신뢰도 낮음
      • 상대방에게 패킷을 보내도 받았는지 체크 안함
    • 비 연결 지향형 프로토콜
    • 데이터를 재전송하지 않는다.
    • 전송된 데이터 일부가 손실될 수 있다.

2계층 : Internet(OSI 7Layer의 3계층 담당) -> IP routing을 함, IP address를 다루는 프로토콜

  • IP routing : source에서 desti로 갈때 어느 router로 갈지 결정하는 것
    • 라우팅 테이블을 통하여 어느 route로 갈지 등록을 해놓는다
    • netstat -r을 했을때 나오는 Destination은 네트워크 IP이다
    • default는 라우터의 IP
    • 한번 ARP로 하면 같은 LAN 상에서는 서로의 MAC주소를 알고 있으니 게이트웨이가 MAC으로 잡힌다
  • ISP(Internet Service Provider) : KT, SK같은 인터넷 공급자들
  • IP
    • 하나의 IP주소에는 Network ID와 Host ID가 존재하고있음
    • Source Address : 4 bytes, 송신 IP
    • Destination Address : 4 bytes, 수신 IP
    • TTL(Time To Leave) : router를 하나씩 거칠때마다 줄어들어 0이 되면 패킷은 버려진다
    • IP는 network 주소와 host주소로 구분된다
      • network주소 : router를 찾는 주소(LAN)
      • host주소 : 찾은 LAN에 소속되어있는 특정 host를 찾는 주소(=컴퓨터)
    • Subneting
    • public IP(공인 IP) -> IP를 돈을 주고 삼(SK, KT 등)
    • private IP
      • Ex) 커피숍에 공인IP하나를 할당해주고 router가 허용하지 않은 대역대 내에서 마음대로 나머지 고객들에게 private IP를 할당함
      • public IP와 private IP를 통신하게 해주는 것이 NAT, NAPT
        • NAT : 사설 IP로 연결되어있는 IP가 외부로 갈경우 매핑되어있는 테이블과 IP를 바꿔 보내고 다시 받으면 테이블의 IP로 다시바꾸고 그것을 다시 ARP로 바꿔 송신받는 기술
        • NAPT(Network Address Port Translation) : NAT에 추가하여 port번호까지 전송
  • BroadCast : LAN상에 있는 모든 사람들에게 패킷을 보내 MAC주소를 알아냄
    • ARP(Address Resolution Protocol)(프로토콜) -> ip를 MAC주소로 변환
      • 같은 LAN 상에있는 호스트들에게 ping요청을 보내 호스트들의 ip와 MAC주소를 매핑시키는 작업
      • Network Interface와 Internet 사이에 있음

1계층 : Network Interface(OSI 7Layer의 1,2계층 담당) : MAC주소

  • Tx에서 쏜 데이터를 Rx에서 받아야함(Serial 통신) -> Data Link 계층
  • NIC(network interface card) : 일반적으로 랜카드라 불림. 네트워크 어댑터
  • MAC(Media Access Control) : NIC의 하드웨어 주소, 앞의 3자리는 제조사 뒤 3자리는 NIC 번호
  • Ethernet(프로토콜) -> lan에서 host를 구별하기위함
    • Destination MAC Address : 6bytes, 패킷을 수신 하는 NIC -> LAN에서의 router
    • Source Mac Address : 6bytes, 패킷을 송신하는 NIC
    • MTU(한번에 보낼수 있는 데이터의 양) -> 1500bytes : 헤더를 제외하면 1350정도

LAN(Local Area Network) : 보통 router를 기준으로 묶여있는 것(같은 router안에서 host를 구분하기위해 MAC주소를 사용함), 같은 LAN상에서 데이터를 보낼때도 이용
WAN(wide Area Network) : IP를 사용

하드웨어 주소 = 물리주소 = MAC주소 = NIC주소 = LAN카드 주소 = 제2계층주소 = 데이터링크계층 주소
이더넷 주소는 하드웨어주소의 일종입니다.

논리주소 = 네트워크 주소 = 제3계층주소 = 프로토콜 주소
IP주소는 논리주소의 한 종류입니다.
