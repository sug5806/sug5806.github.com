이진 트리 운행(Traversals)

트리를 다루기 위해서는 트리를 운행하기 위한 메커니즘 필요한데 이것이 여기서 다룰 내용입니다. 트리의 모든 노드들은 방문하는 과정을 트리 운행이라 합니다. 각 노드들은 한 번만 처리되겠지만, 여러 번 방문을 받을 수 있습니다.

운행 방법(Traversal Possibilities)

운행 방법을 아래와 같이 3가지로 나타낼 수 있습니다.

  • 전위 운행법(Preorder)
  • 중위 운행법(Inorder)
  • 후위 운행법(Postorder)

여기에 순서를 따르지 않는 다른 운행법을 추가합니다.

  • 레벨 순서 운행법(Level Order Traversal): 이 함수는 너비 우선 탐색에서 영향을 받았습니다(그래프 알고리즘의 BFS).

전위 운행법(Preorder)

전위 운행법에서는 각 노드의 자신의 좌, 우측 하위 트리보다 먼저 처리됩니다.

  • root를 방문합니다.
  • 다시 전위 운행법으로 왼쪽 하위 트리를 운행합니다.
  • 전위 운행법으로 오른쪽 하위 트리를 운행합니다.

위의 트리에서는 1, 2, 4, 5, 3, 6, 7 순서로 처리됩니다.

중위 운행법(Inorder)

중위 운행법에서 root는 오른쪽 하위 트리와 왼쪽 하위 트리 사이에 방문하게 됩니다. 중위 운행법은 다음과 같이 정의됩니다.

  • 왼쪽 하위트리를 탐색합니다.
  • root를 처리합니다.
  • 오른쪽 하위 트리를 중위 운행법으로 탐색합니다.

위의 트리에서는 4, 2, 5, 1, 6, 3, 7 순서로 처리됩니다.

후위 운행법(Postorder)

후위 운행에서 root는 양쪽 하위 트리가 처리된 이후 처리됩니다. 후위 운행법은 다음과 같이 정의합니다.

  • 왼쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • 오른쪽 하위 트리를 후위 운행법으로 탐색합니다.
  • root를 처리합니다.

앞의 예제 트리의 노드들은 4, 5, 2, 6, 7, 3, 1 순서로 처리됩니다.

트리란 무엇인가?

트리란 연결 리스트와 유사한 자료 구조지만 연결 리스트가 단순히 다음 노드를 가리키는 선형이라면 트리느 각 노드가 여러 개의 노드를 가리키는 비선형입니다. 트리는 비선형 자료 구조의 대표적인 예이면서, 그래픽 형태로 계층적 구조의 특성을 표현하는 방법입니다. 트리의 추상 자료형(ADT)에서는 요소들의 순서가 중요하지 않습니다. 만일 순서가 중요하다면 연결 리스트, 스택, 큐와 같은 선형 자료 구조를 사용해야 합니다.

용어 정의

  • 트리의 root는 부모를 가지지 않습니다. 트리에는 최대 하나의 root노드가 있을 수 있습니다.
  • edge는 부모와 자식을 잇는 연결 선입니다(그림에서의 모든 연결 선).
  • 자식이 없는 노드를 leaf 노드라고 부릅니다(E, J, K, H, I).
  • 같은 부모의 자식을 siblings라고 부릅니다(E, J, K, H, I).
  • 노드의 depth는 root에서 해당 노드까지 경로(path)의 길이입니다.(G의 depth는 2(A-C-G)입니다)
  • 트리의 height는 트리 내 모든 노드들 중 높이가 가장 큰 값이고 트리의 depth는 모든 노드들 중에서 depth가 가장 큰 값입니다.
  • 지정된 depth에 위치한 모든 노드들을 트리의 레벨(level)이라 부릅니다. (예에서 B, C 그리고 D는 같은 레벨입니다). Root 노드의 레벨은 0입니다.

힙(Heap)이란

힙에 기본적인 조건은 노드의 값이 자신의 자식들의 값보다 반드시 크거나 같아야(혹은 작거나 같아야)한다는 것입니다. 이것을 '힙의 특성(heap property)'라고 합니다. 또한 힙은 추가적인 특성들도 가지는데 모든 leaf들은 h>0 일때 (h은 완전 이진 트리에서의 트리의 높이) h 또는 h-1 레벨에 존재합니다. 힙은 완전 이진 트리를 형성해야 합니다.

힙의 종류

  • 최소힙(Min heap): 노드의 값이 자신의 자식 노드의 값보다 작거나 같습니다.

  • 최대힙(Max heap): 노드의 값이 자신의 자식 노드의 값보다 크거나 같습니다.

힙의 추상화(Heap ADT)

힙의 주요 연산들

  • push(data): 힙에 데이터를 넣습니다
  • pop(): 힙에서 데이터를 삭제하고 반환합니다

힙의 보조 연산들

  • is_empty(): 힙이 비어있는지 여부를 반환합니다
  • is_full(): 힙이 가득 차 있는지 여부를 반환합니다
  • top(): 다음번에 삭제할 데이터를 반환합니다.
