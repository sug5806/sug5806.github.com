2019-08-16 트리(Tree)

2019/08/16 카테고리 자료구조

개인적으로 공부한 것을 정리하였습니다.

트리란 무엇인가?

트리란 연결 리스트와 유사한 자료 구조지만 연결 리스트가 단순히 다음 노드를 가리키는 선형이라면 트리느 각 노드가 여러 개의 노드를 가리키는 비선형입니다. 트리는 비선형 자료 구조의 대표적인 예이면서, 그래픽 형태로 계층적 구조의 특성을 표현하는 방법입니다. 트리의 추상 자료형(ADT)에서는 요소들의 순서가 중요하지 않습니다. 만일 순서가 중요하다면 연결 리스트, 스택, 큐와 같은 선형 자료 구조를 사용해야 합니다.

용어 정의

  • 트리의 root는 부모를 가지지 않습니다. 트리에는 최대 하나의 root노드가 있을 수 있습니다.
  • edge는 부모와 자식을 잇는 연결 선입니다(그림에서의 모든 연결 선).
  • 자식이 없는 노드를 leaf 노드라고 부릅니다(E, J, K, H, I).
  • 같은 부모의 자식을 siblings라고 부릅니다(E, J, K, H, I).
  • 노드의 depth는 root에서 해당 노드까지 경로(path)의 길이입니다.(G의 depth는 2(A-C-G)입니다)
  • 트리의 height는 트리 내 모든 노드들 중 높이가 가장 큰 값이고 트리의 depth는 모든 노드들 중에서 depth가 가장 큰 값입니다.
  • 지정된 depth에 위치한 모든 노드들을 트리의 레벨(level)이라 부릅니다. (예에서 B, C 그리고 D는 같은 레벨입니다). Root 노드의 레벨은 0입니다.
